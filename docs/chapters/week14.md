## EASY BOX 내용

**핵심 비유**: "논문은 '내가 이런 문제를 이렇게 풀었고, 이렇게 확인했어요'를 남에게 정확히 설명하는 글이다."

**예시 리스트**:
- 좋은 논문 = 레시피처럼 재현 가능해야 함 (다른 사람도 따라 할 수 있어야 함)
- 수식은 "모호함을 없애는 도구" — 말로 하면 헷갈리는 것을 수식으로 정확히
- AI에서: 논문에서 P(y|x), loss function, evaluation metric이 전부 여기서 배운 확률로 표현됨

## 섹션 구성

### 섹션 1: 논문의 기본 구조

**아이콘**: `📄`  
**제목**: `AI/ML 논문의 표준 구조`

**단계별 설명**:
1. **Abstract (초록)**: 논문 전체를 100~200단어로 요약. 문제, 방법, 결과를 포함
2. **Introduction (서론)**: 문제 정의, 기존 방법의 한계, 기여(contribution) 명시
3. **Related Work (관련 연구)**: 기존 연구와 차별점 설명
4. **Method (방법론)**: 제안하는 방법의 수학적 정의. 여기서 P(y|x), loss 등이 등장
5. **Experiments (실험)**: 실험 설정, 비교 대상(baseline), 지표(metric) 설명
6. **Results & Analysis (결과)**: 숫자로 성능 비교, ablation study
7. **Conclusion (결론)**: 한계(limitation)와 향후 연구(future work)

---

### 섹션 2: 논문 속 확률 표기 읽기

**아이콘**: `🔍`  
**제목**: `논문에서 자주 보이는 수학 표기`

**표 (확률 표기 패턴)**:

| 표기 | 의미 | 어느 논문에서 |
|------|------|------------|
| `P(y|x; θ)` | 파라미터 θ를 가진 모델의 조건부 확률 | 분류, 언어모델 |
| `q_φ(z|x)` | φ가 파라미터인 인코더의 근사 posterior | VAE |
| `p_θ(x|z)` | θ가 파라미터인 디코더의 likelihood | VAE, 생성모델 |
| `L(θ)` | 파라미터 θ에 대한 손실함수 | 모든 ML 논문 |
| `argmin_θ L(θ)` | L을 최소화하는 θ 찾기 | 최적화 목표 |
| `KL(q‖p)` | q와 p의 KL divergence | VAE ELBO |
| `E_x[f(x)]` | f(x)의 x에 대한 기대값 | 기대 손실 |

### 섹션 3: 논문 읽기 전략

**아이콘**: `📖`  
**제목**: `효율적인 논문 읽기 전략`

**단계별 설명**:
1. **제목·Abstract 먼저**: "이 논문이 무엇을 해결하려는가?" 파악
2. **Figure·Table 훑기**: 핵심 결과를 시각적으로 파악. 이게 논문의 "주장"
3. **Introduction 읽기**: 문제 정의와 기여 파악
4. **Method 수식 읽기**: 로스함수, 파라미터, 최적화 목표 확인
5. **Experiments 확인**: baseline과 제안 방법 비교, ablation
6. **결론·한계 확인**: 어디까지 됐고 어디서 안 됐는지

**callout (tip)**:
- 제목: `처음에는 식 전체를 이해 못해도 된다`
- 내용: 수식에서 찾아야 하는 것:
  1. 무엇을 최적화하는가? (loss/objective)
  2. 최적화하는 대상은? (θ, φ, w)
  3. 데이터는 어떻게 들어가는가? (x, y 기호 확인)

### 섹션 4: 연구 제안서 작성

**아이콘**: `✍️`  
**제목**: `내 연구문제 수학적으로 표현하기`

**설명**: 논문을 쓰기 전에 자신의 연구 문제를 명확하게 정의해야 합니다.

**단계별 설명**:
1. **문제 정의**: "무엇을 입력으로 받아 무엇을 출력하는가?" — `f: X → Y` 형식으로
2. **데이터 정의**: 훈련 데이터 `D = {(xᵢ, yᵢ)}_{i=1}^n`
3. **모델 정의**: `P_θ(y|x) = ...` 형식으로 모델 명시
4. **손실함수**: 최소화할 목적함수 `min_θ L(θ)` 정의
5. **평가 지표**: accuracy, F1, BLEU, ECE 등 측정 방법 명시

## 퀴즈

**질문**: 논문에서 "argmin_θ L(θ)"가 의미하는 것은?

| 선택지 | 정답 여부 |
|--------|----------|
| (a) 손실함수 L의 최대값 | ❌ |
| (b) L을 가장 크게 만드는 파라미터 θ | ❌ |
| (c) 손실함수 L을 최소화하는 파라미터 θ를 찾는 것 | ✅ |
| (d) θ의 기대값 계산 | ❌ |

**정답 위치**: (c)
