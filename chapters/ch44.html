LLM Alignment 이론 — RLHF 수학과 DPO 도출

RLHF의 수학적 정식화 완전 유도 — Bradley-Terry 보상 모델, KL 발산 정규화 RL의 Closed-Form 해, 그리고 DPO가 왜 RLHF와 동치인지. NeurIPS/ICML 수준 Alignment 논문 독해를 위한 필수 이론.

🧒 초등학생 한 줄 설명

새 직원을 훈련시킬 때, '하루 일당을 주는 대신 계속 일하게 하면 무조건 나쁜 짓도 한다.' RLHF는 '좋은 행동에 보너스, 나쁜 행동에 패널티'를 주되, '원래 성격(기준 모델)에서 너무 멀어지면 안 된다'는 KL 제약을 건다. DPO는 이 최적해가 수식으로 표현되므로, 보상 모델 없이 직접 그 해로 가는 길이다.

  • KL 발산 없이 보상만 최대화: 점수가 높은 "완벽한 문장" 하나를 무한 반복 출력 (보상 해킹)
  • KL 제약 추가: 기준 모델과 비슷하게 유지하면서 보상 최대화 → 균형 잡힌 최적화
  • DPO의 통찰: KL-제약 RL의 해 \(\pi^*(y|x) \propto \pi_\text{ref}(y|x)\cdot\exp(r(x,y)/\beta)\) → 이를 역이용
  • AI에서: Claude (Constitutional AI), InstructGPT, LLaMA 2 Chat 모두 이 이론 기반

📐KL-제약 강화학습 — 완전 수학적 정식화

RLHF는 다음의 최적화 문제로 정식화됩니다: 보상 함수를 최대화하되, 기준 정책(reference policy)과의 KL 발산을 \(\beta\)로 제한합니다.

RLHF 최적화 문제 (KL-제약 RL)
\[\pi^* = \arg\max_{\pi} \mathbb{E}_{x\sim\mathcal{D},\, y\sim\pi(\cdot|x)}\!\left[r(x,y)\right] - \beta\, \mathbb{E}_{x\sim\mathcal{D}}\!\left[D_{\text{KL}}\!\left(\pi(\cdot|x) \,\|\, \pi_{\text{ref}}(\cdot|x)\right)\right]\]
\(r(x,y)\): 보상 함수 — 보상 모델이 출력하는 점수
\(\pi_\text{ref}\): 기준 정책 — SFT(Supervised Fine-Tuning) 모델
\(\beta\): KL 패널티 강도. \(\beta=0\)이면 보상만 최대화(보상 해킹 위험), \(\beta\to\infty\)이면 \(\pi\to\pi_\text{ref}\)
Constrained MDP의 특수 케이스
KL-제약 RL의 Closed-Form 최적해 (완전 유도)
\[\pi^*(y \mid x) = \frac{\pi_{\text{ref}}(y \mid x) \cdot \exp\!\left(r(x,y)/\beta\right)}{Z(x)} \quad\text{where}\quad Z(x) = \sum_y \pi_{\text{ref}}(y \mid x)\cdot e^{r(x,y)/\beta}\]
유도: 목적함수에 \(\partial/\partial\pi = 0\) → 라그랑주 승수법으로 직접 해 도출
\(Z(x)\): 정규화 상수(분배 함수). 직접 계산 불가능(모든 \(y\)에 대한 합)
해석: 보상이 높은 응답은 기준 정책 확률보다 \(\exp(r/\beta)\)배 높아짐
이 해가 존재한다는 것 = RLHF 문제의 해가 항상 이 형태
💡

DPO의 핵심 통찰: 위 최적해를 역으로 표현하면 \(r(x,y) = \beta\cdot\log[\pi^*(y|x)/\pi_\text{ref}(y|x)] + \beta\cdot\log Z(x)\). 보상 \(r(x,y)\)를 \(\pi^*\)의 함수로 표현할 수 있다! 이 관계를 Bradley-Terry 손실에 대입하면 보상 모델 없이 \(\pi\)를 직접 최적화하는 DPO가 도출됩니다.

⚖️쌍대 선호도에서 보상 점수 학습하기

인간이 "A보다 B가 더 좋다"고 말할 때, 이 선호도를 확률 모델로 표현한 것이 Bradley-Terry 모델입니다.

Bradley-Terry 선호 모델
\[P(y_w \succ y_l \mid x) = \sigma(r(x, y_w) - r(x, y_l)) = \frac{e^{r(x,y_w)}}{e^{r(x,y_w)} + e^{r(x,y_l)}}\]
\(y_w \succ y_l\): "\(y_w\)가 \(y_l\)보다 선호됨" (인간이 선택)
\(\sigma\): 시그모이드 함수. \(r\) 차이가 클수록 확률이 1에 가까워짐
보상 모델 학습 = 이 확률을 최대화하는 MLE = 이진 교차 엔트로피
확률적 선호(deterministic 아님): 같은 비교에서도 노이즈 존재
DPO 완전 도출 — 보상을 π로 치환
\[\mathcal{L}_{\text{DPO}} = -\mathbb{E}\!\left[\log\sigma\!\left(\beta\log\frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta\log\frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]\]
핵심 치환: \(r(x,y) = \beta\cdot\log[\pi_\theta(y|x)/\pi_\text{ref}(y|x)] + \beta\cdot\log Z(x)\)
\(\log Z(x)\)는 \(y_w\), \(y_l\) 비교에서 동일하게 상쇄 → 사라짐
결과: 보상 모델 \(r_\phi\) 없이 \(\pi_\theta\)만으로 DPO 학습 가능
이것이 DPO가 "보상 모델 없이 RLHF와 동치"인 수학적 이유

RLAIF vs RLHF: Lee et al., 2023에서 RLAIF가 RLHF와 비슷한 성능을 내면서 인간 어노테이션 없이 확장 가능함을 보임. 현재 대부분의 대형 AI 기업이 활용.

⚠️보상 해킹과 Goodhart's Law

"보상 측정 지표가 목표가 되는 순간, 그것은 더 이상 좋은 측정 지표가 아니다" — Goodhart's Law. RLHF에서도 동일한 문제가 발생합니다.

보상 해킹의 수학적 이해
\[\mathbb{E}[r_{\text{true}}] \leq \mathbb{E}[r_{\hat{\phi}}] \quad\text{단,}\quad \mathbb{E}[r_{\hat{\phi}}] \nearrow \text{ as training} \nearrow\]
\(r_\text{true}\): 진짜 인간의 선호, \(r_{\hat{\phi}}\): 학습된 보상 모델의 점수
보상 모델은 훈련 데이터 분포 밖에서 오버핏 → 진짜 보상과 분리됨
학습이 길어질수록 보상 모델 점수는 오르지만 실제 품질은 떨어질 수 있음
해결: KL 제약(\(\beta\)), 더 많은 인간 피드백, 보상 앙상블

RLHF 실패 모드와 해결책

실패 모드원인해결책
보상 해킹보상 모델 과신KL 제약 강화, 보상 앙상블
응답 길이 편향긴 응답 = 높은 점수길이 패널티, 길이 정규화
