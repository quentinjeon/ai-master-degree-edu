LLM 평가 방법론

"이 AI가 얼마나 좋은가?"를 측정하는 법 — BLEU의 한계부터 LLM-as-a-Judge까지, 논문 결과 섹션을 읽기 위한 필수 지식.

🧒 초등학생 한 줄 설명

"학생 글짓기를 채점하는 방법이다. ①단순히 표준 답안과 단어가 얼마나 겹치는지 세는 법(BLEU/ROUGE), ②전문가 교사가 직접 읽고 점수를 주는 법(인간 평가), ③AI 교사가 대신 읽고 채점하는 법(LLM-as-a-Judge)"

  • BLEU가 높아도 내용이 이상한 경우: "The cat sat on the mat" → "The mat sat on the cat" (단어는 같지만 의미 반전)
  • 인간 평가의 한계: 비용 높음, 주관적, 느림
  • LLM-as-a-Judge: GPT-4가 채점자 역할 → 인간 평가와 상관관계 높음
  • AI에서: Chatbot Arena, LMSYS, AlpacaEval이 LLM-as-a-Judge 기반

📏 BLEU·ROUGE — 단어 겹침 기반 평가의 한계

BLEU는 기계 번역 평가를 위해 고안되었지만, 개방형 텍스트 생성 평가에는 한계가 있습니다. 동의어, 문체 다양성, 의미적 정확성을 측정하지 못합니다.

BLEU 점수 (Papineni et al., 2002)
\[\text{BLEU} = BP \cdot \exp\!\left(\sum_{n=1}^{N} w_n \log p_n\right) \quad\text{where}\quad BP = \begin{cases} 1 & c \geq r \\ e^{1-r/c} & c < r \end{cases}\]
\(p_n\): n-gram 정밀도(생성 텍스트 중 참조 텍스트에 나온 n-gram 비율)
\(w_n = 1/N\): n-gram 가중치(보통 N=4로 동일 가중치)
\(BP\): 짧은 생성에 불이익 주는 Brevity Penalty
\(c\): 생성 텍스트 길이, \(r\): 참조 텍스트 길이
ROUGE-L (Lin, 2004)
\[\text{ROUGE-L} = \frac{(1+\beta^2) \cdot R_{lcs} \cdot P_{lcs}}{R_{lcs} + \beta^2 P_{lcs}} \quad\text{where}\quad R_{lcs} = \frac{LCS(X,Y)}{m},\; P_{lcs} = \frac{LCS(X,Y)}{n}\]
\(LCS(X,Y)\): 참조(X)와 생성(Y) 텍스트의 최장 공통 부분 수열 길이
\(m\): 참조 텍스트 길이, \(n\): 생성 텍스트 길이
ROUGE-L: 요약 평가에 자주 사용 (순서까지 고려)
\(\beta=1.2\) (재현율 강조): 요약 분야 표준

⚖️ LLM-as-a-Judge — AI가 AI를 평가한다

GPT-4 같은 강력한 LLM을 평가자로 사용하는 방법입니다. 정해진 기준(Rubric)을 프롬프트로 제공하고, LLM이 점수 + 설명을 생성합니다. 인간 평가와의 상관관계가 BLEU보다 훨씬 높습니다.

LLM-as-a-Judge 평가 과정

1
평가 기준 정의(Rubric) — 정확성, 유창성, 관련성 등 채점 항목 설계
2
평가 프롬프트 설계 — Rubric + 평가 대상 텍스트를 LLM에게 전달
3
LLM 실행(GPT-4, Claude 등) — 점수와 근거 설명 생성
4
집계 & 분석 — 다수 평가 결과 평균, 신뢰도 계산
🔑 LLM-as-a-Judge 편향 유형 (Zheng et al., 2023)
  • 위치 편향: 먼저 제시된 응답 선호
  • 자기 편향: 평가자 LLM 동일 계열 모델에 유리
  • 길이 편향: 더 긴 응답 선호
  • 포맷 편향: 마크다운 형식 선호
  • 해결책: 위치 스왑, 다수 평가자, CoT 요구

🏟️ Chatbot Arena — 절대 점수 대신 상대적 순위

LMSYS Chatbot Arena는 두 모델을 무작위로 A/B 테스트하여 사용자가 더 좋은 응답을 선택하게 합니다. Elo 점수로 모델 순위를 매깁니다.

Elo 레이팅 업데이트
\[R_A' = R_A + K \cdot (S_A - E_A) \quad\text{where}\quad E_A = \frac{1}{1 + 10^{(R_B - R_A)/400}}\]
\(R_A, R_B\): 모델 A, B의 현재 Elo 점수
\(E_A\): 모델 A가 이길 기대 확률(로지스틱 함수)
\(S_A\): 실제 결과(이기면 1, 비기면 0.5, 지면 0)
\(K\): 업데이트 속도(보통 K=4~32)
현재 LLM 순위 확인 LMSYS Chatbot Arena (lmarena.ai) — 수백만 건의 인간 투표로 만들어진 가장 신뢰할 수 있는 LLM 벤치마크.
