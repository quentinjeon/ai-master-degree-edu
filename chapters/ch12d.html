"생각하고 나서 답하는" AI — CoT가 LLM의 수학 실력을 2배로 높이는 원리, 그리고 DeepSeek-R1을 만든 GRPO.

🧒 초등학생 한 줄 설명

"수학 시험에서 답만 적는 학생보다 풀이 과정을 쓰는 학생이 더 정확하다. CoT는 LLM에게 '답 전에 풀이 과정부터 쓰게' 하는 방법이다."

  • 직접 답변: "17×24 = ?" → 중간 단계 없이 바로 답하면 오류 가능성이 높다
  • CoT 적용: "17×24 = 17×20 + 17×4 = 340 + 68 = 408" → 단계별 검증으로 정확도 상승
  • Self-Consistency: 같은 문제를 10번 풀어 가장 많이 나온 답을 선택 (다수결)
  • AI에서: GPT-o1, o3, DeepSeek-R1 등 추론 모델이 CoT + GRPO 조합으로 탄생했다

🧠 Chain of Thought — 단계별로 생각하면 더 잘 맞춘다

CoT(Wei et al., 2022)는 LLM이 최종 답변 전에 중간 추론 단계를 생성하도록 유도합니다. "단계별로 생각해보자"(Let's think step by step)라는 문구 하나로 수학·논리 문제 정확도를 크게 높입니다.

CoT 생성 확률
\[P(a \mid x) = \sum_{r} P(a \mid x, r) \cdot P(r \mid x)\]
\(x\): 입력 질문  |  \(r\): 추론 체인(reasoning chain)  |  \(a\): 최종 답
직접 생성: \(P(a \mid x)\) — 중간 단계 없이 바로 답을 예측
CoT: \(r\)을 먼저 생성하고, 그 맥락에서 \(a\)를 예측 → 더 정확한 조건부 확률
마치 조건부 확률의 주변화처럼 더 많은 정보를 활용하여 답을 도출

CoT 변형 기법

기법방법장점
Zero-shot CoT"Let's think step by step" 추가예시 불필요
Few-shot CoT풀이 과정 포함 예시 3~5개 제공더 높은 정확도
Self-Consistency다수결 투표신뢰도 향상
ToT (Tree of Thoughts)여러 추론 경로 탐색복잡한 계획 문제

🗳 Self-Consistency — 다수결로 더 정확하게

Self-Consistency(Wang et al., 2022)는 동일한 프롬프트를 여러 번 샘플링하여 가장 자주 나온 답을 선택합니다. 단일 추론 경로에 의존하는 greedy decoding의 한계를 극복합니다.

Self-Consistency 다수결
\[\hat{a} = \arg\max_{a} \sum_{i=1}^{N} \mathbf{1}\!\left[a_i = a\right] \quad \text{where each } a_i \sim P(\cdot \mid x, r_i),\; r_i \sim P(\cdot \mid x)\]
\(N\)번 샘플링하여 가장 많이 등장한 답 \(\hat{a}\)를 최종 선택
\(N\)이 클수록 정확도 향상 (보통 \(N=40\)이면 수렴)
단점: 추론 비용 \(N\)배 증가 (비용-정확도 트레이드오프)
고성능이 필요할 때만 사용 — 실시간 응답보다는 정확도 우선 시스템에 적합

🚀 GRPO — DeepSeek-R1의 핵심 알고리즘

GRPO(Group Relative Policy Optimization, Shao et al., 2024)는 DeepSeek-R1을 만든 핵심 알고리즘입니다. 같은 질문에 대해 여러 응답을 생성하고, 그룹 내 상대적 보상을 사용하여 PPO의 가치 함수(Value Network) 없이 학습합니다.

GRPO 목적함수 (Shao et al., 2024)
\[J_{\text{GRPO}}(\theta) = \mathbb{E}_{q \sim \mathcal{D},\, \{o_i\}_{i=1}^G \sim \pi_{\theta_{\text{old}}}}\!\left[\frac{1}{G}\sum_{i=1}^G \min\!\left(\rho_i A_i,\; \text{clip}(\rho_i, 1\pm\epsilon)A_i\right) - \beta D_{\text{KL}}\right]\]
\(q\): 질문  |  \(G\): 그룹 크기(한 번에 생성하는 응답 수, 보통 8~16)
\(o_i\): \(i\)번째 응답  |  \(\rho_i = \pi_\theta(o_i \mid q) / \pi_{\theta_\text{old}}(o_i \mid q)\): 중요도 비율
\(A_i\): 어드밴티지 — 그룹 내 상대적 보상으로 계산
\(\text{clip}(\cdot, 1\pm\epsilon)\): PPO 클리핑 — 너무 큰 업데이트 방지  |  \(\beta D_\text{KL}\): KL 정규화 항
GRPO 어드밴티지 계산
\[A_i = \frac{r_i - \text{mean}(\{r_j\}_{j=1}^G)}{\text{std}(\{r_j\}_{j=1}^G)}\]
\(r_i\): \(i\)번째 응답의 보상 (정답이면 +1, 오답이면 0 또는 −1)
그룹 내 평균을 빼고 표준편차로 나눔 → 상대적 우열만 신호로 사용
가치 함수(Critic Network) 학습이 불필요 → PPO보다 단순하고 안정적
DeepSeek-R1의 수학 추론 능력이 이 방법으로 크게 향상됨
📌 DeepSeek-R1과 GRPO DeepSeek-R1(2025)은 GRPO를 사용하여 수학·코딩 문제에서 OpenAI o1에 근접하는 성능을 달성했습니다. 핵심은 "생각 과정(Thinking)"을 출력하도록 훈련 + 결과 기반 보상(정답=1, 오답=0)만 사용하는 단순한 GRPO. 별도의 Critic Network 없이 그룹 내 비교만으로 강화학습을 안정적으로 수행합니다.

🔀 MoE — 파라미터는 많지만 쓰는 건 일부만

MoE(Mixture of Experts)는 LLM의 FFN 레이어를 여러 "전문가" 네트워크로 교체하고, 각 토큰마다 일부 전문가만 활성화하는 구조입니다. 전체 파라미터는 매우 크지만, 추론 시 실제 연산량은 활성화된 전문가 수에만 비례합니다.

MoE 라우팅
\[\text{MoE}(x) = \sum_{i \in \text{top-}k} g_i(x) \cdot E_i(x) \quad\text{where}\quad g(x) = \text{softmax}(W_g x)\]
\(E_i(x)\): \(i\)번째 전문가 네트워크의 출력
\(g_i(x)\): \(i\)번째 전문가의 게이팅 가중치 (상위 \(k\)개만 활성화)
Mistral-8×7B: 8개 전문가 중 2개 활성화 → 실제 연산량 = 13B 파라미터 수준, 용량 = 47B
GPT-4는 추정상 16개 전문가 × 111B 파라미터 MoE 구조
