Transformer Attention 완전 유도

Scaled Dot-Product Attention, Multi-Head, Positional Encoding의 수학. "Attention is All You Need" 논문을 수식 수준으로 읽습니다.

🧒 초등학생 한 줄 설명

도서관에서 책을 찾을 때 "쿼리(찾는 키워드)"로 "키(책 제목)"를 검색하면 관련도(attention score)가 나오고, 그 점수에 따라 "값(책 내용)"을 가져온다 — 이것이 Attention의 전부다.

  • Query: 현재 처리하는 단어 ("the cat sat") — 지금 뭘 찾고 있는지
  • Key / Value: 모든 단어의 검색 표현(Key)과 실제 내용(Value) — 가중치 합으로 문맥 정보를 가져옴
  • AI에서: ChatGPT, BERT, GPT-4 모두 이 메커니즘을 수십 레이어 쌓은 것 — 수식 수준으로 이해하면 어떤 논문도 읽을 수 있다

⊙Scaled Dot-Product Attention

Attention 메커니즘은 Query(Q), Key(K), Value(V) 행렬을 사용하여 각 위치에서 다른 모든 위치의 정보를 가중합합니다. Scaling이 없으면 softmax의 그라디언트가 소멸합니다.

SCALED DOT-PRODUCT ATTENTION
\[ \mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V \]
\(Q \in \mathbb{R}^{n \times d_k}\): 쿼리 행렬 — \(n\)=시퀀스 길이, \(d_k\)=키 차원
\(K \in \mathbb{R}^{m \times d_k}\): 키 행렬 — \(m\)개 위치의 검색 가능한 표현
\(V \in \mathbb{R}^{m \times d_v}\): 값 행렬 — 각 위치의 실제 정보
\(\sqrt{d_k}\): 스케일링 인자 — \(QK^\top\)의 분산이 \(d_k\)에 비례 → \(1/\sqrt{d_k}\)로 표준화
결과: \(\mathbb{R}^{n \times d_v}\) — 각 쿼리 위치에서의 문맥화된 표현

Scaling이 필요한 이유 — 분산 분석

내적 \(q_i \cdot k_j\)의 분산은 차원 \(d_k\)에 비례하여 커집니다. 스케일 없이 softmax를 적용하면 특정 위치에 확률이 극단적으로 집중되어 그라디언트가 거의 0이 됩니다.

SCALING NECESSITY — VARIANCE ANALYSIS
\[ \mathrm{Var}[q_i \cdot k_j] = d_k \cdot \mathrm{Var}[q] \cdot \mathrm{Var}[k] = d_k \quad\Rightarrow\quad \frac{q_i \cdot k_j}{\sqrt{d_k}} \sim \mathcal{N}(0,1) \text{ (근사)} \]
\(q\)와 \(k\)의 각 차원이 \(\mathcal{N}(0,1)\)이면 내적의 분산 \(= d_k\)
\(\sqrt{d_k}\)로 나누면 분산 \(= 1\) → softmax 포화(saturation) 방지
\(d_k\)가 크면 스케일 없을 때 softmax가 극단적으로 집중됨 → 그라디언트 소멸
원논문: "We suspect that for large values of \(d_k\), the dot products grow large in magnitude, pushing the softmax function into regions where it has extremely small gradients."
📄

직관적 이해: \(d_k = 64\)이면 내적 분산이 64, 표준편차가 8이 됩니다. \(\sqrt{64} = 8\)로 나누어야 분산이 1이 되어 softmax가 안정적으로 작동합니다. 이것이 원논문에서 \(d_k = 64\) (=512/8)를 선택한 이유이기도 합니다.

HMulti-Head Attention

단일 Attention 대신 \(h\)개의 헤드로 서로 다른 서브공간에서 동시에 Attention을 계산합니다. 각 헤드는 다른 종류의 관계(구문적, 의미적, 장거리 의존성 등)를 독립적으로 포착합니다.

MULTI-HEAD ATTENTION
\[ \mathrm{MultiHead}(Q,K,V) = \mathrm{Concat}(\text{head}_1, \ldots, \text{head}_h)\,W^O \] \[ \text{where} \quad \text{head}_i = \mathrm{Attention}(QW_i^Q,\; KW_i^K,\; VW_i^V) \]
\(W_i^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}\): \(i\)번째 헤드의 쿼리 투영 행렬
\(W_i^K \in \mathbb{R}^{d_{\text{model}} \times d_k}\), \(W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}\): 키, 값 투영 (\(d_k = d_v = d_{\text{model}}/h\))
\(W^O \in \mathbb{R}^{h d_v \times d_{\text{model}}}\): 출력 투영 행렬
\(h\)개 헤드를 연결(Concat) 후 선형 투영 → \(d_{\text{model}}\) 차원 출력
계산 비용: 단일 Attention과 동일 — 각 헤드 차원이 \(1/h\)로 줄어들기 때문

Transformer 원논문 하이퍼파라미터 (Vaswani et al., 2017)

파라미터값의미
\(d_{\text{model}}\)512모델 전체 차원 (Embedding + 모든 레이어)
\(h\)8헤드 수 — 8가지 서브공간에서 동시 Attention
\(d_k = d_v\)64헤드당 차원 (= 512 / 8)
\(d_{\text{ff}}\)2048FFN 내부 차원 (= 4 × d_model)
\(N\) (레이어)6인코더/디코더 레이어 수
Multi-Head의 장점
  • 각 헤드가 다른 유형의 관계를 학습
  • 일부 헤드: 인접 단어 의존성 (구문)
  • 일부 헤드: 장거리 의존성 (의미)
  • 앙상블 효과 — 단일 Attention보다 강인
파라미터 수 계산 (d_model=512, h=8)
  • 헤드당: \(W^Q, W^K, W^V\) 각 512×64 = 32768
  • 8헤드 × 3행렬 × 32768 ≈ 786K
  • 출력 투영 \(W^O\): 512×512 = 262K
  • 한 층 총합 ≈ 1.05M 파라미터

📍Positional Encoding

Attention 자체는 입력 순서에 무관합니다 — 단어 순서를 바꿔도 동일한 결과가 나옵니다. 위치 정보를 Embedding에 더하는 것이 Positional Encoding입니다.

SINUSOIDAL POSITIONAL ENCODING
\[ PE_{(pos,\,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right) \qquad PE_{(pos,\,2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right) \]
\(pos\): 시퀀스에서의 위치 (0부터 시작)
\(i\): 차원 인덱스 (\(0 \leq i < d_{\text{model}}/2\))
각기 다른 주파수의 sin/cos — 멀리 떨어진 위치는 서로 다른 패턴을 가짐
장점: 학습 시 보지 못한 긴 시퀀스도 외삽(extrapolation) 가능
최신: 대부분 학습 가능한 Positional Embedding 또는 RoPE를 사용
🔄

RoPE (Rotary Position Embedding) (Su et al., 2021): 위치를 복소수 회전으로 표현합니다. 상대 위치 정보를 Attention 점수 자체에 직접 주입 → 절대 위치가 아닌 상대 위치를 자연스럽게 인코딩. LLaMA, GPT-NeoX, Mistral 등 최신 LLM에서 채택.

⚠️

흔한 오해: Positional Encoding은 Embedding에 더하는 것이지 연결(concat)하는 것이 아닙니다. 입력 \(X\)의 각 위치 벡터에 \(PE_{pos}\)를 원소별로 더합니다: \(\tilde{x}_{pos} = x_{pos} + PE_{pos}\). 따라서 차원이 변하지 않습니다.

🏗Transformer 블록의 완전한 구조

Transformer 인코더 블록은 Self-Attention과 Feed-Forward Network(FFN)를 Residual Connection + LayerNorm으로 연결합니다. 디코더는 추가로 Causal Mask와 Cross-Attention을 포함합니다.

TRANSFORMER ENCODER BLOCK
\[ \begin{aligned} \mathbf{Z} &= \mathrm{LayerNorm}(\mathbf{X} + \mathrm{MultiHead}(\mathbf{X},\mathbf{X},\mathbf{X})) \\ \mathbf{Y} &= \mathrm{LayerNorm}(\mathbf{Z} + \mathrm{FFN}(\mathbf{Z})) \\ \mathrm{FFN}(\mathbf{z}) &= \max(0,\, \mathbf{z}W_1 + b_1)W_2 + b_2 \end{aligned} \]
Self-Attention: \(Q = K = V = \mathbf{X}\) — 자기 자신에 대한 Attention
Residual connection: \(\mathbf{X}\)를 더해 그라디언트 흐름 보장 (vanishing gradient 방지)
LayerNorm: 각 위치의 특징 차원으로 정규화 — BatchNorm과 달리 배치 크기에 독립적
FFN: 2레이어 MLP — 각 위치에 독립적으로 적용 (위치 간 정보 교환 없음)
\(W_1 \in \mathbb{R}^{d_{\text{model}} \times d_{ff}}\), \(W_2 \in \mathbb{R}^{d_{ff} \times d_{\text{model}}}\)
MASKED SELF-ATTENTION — CAUSAL MASK
\[ \mathrm{score}_{ij} = \begin{cases} \dfrac{q_i \cdot k_j}{\sqrt{d_k}} & j \leq i \\[8pt] -\infty & j > i \end{cases} \]
자기회귀 생성: 위치 \(i\)는 위치 \(j \leq i\)의 정보만 볼 수 있음 (미래 정보 차단)
\(j > i\)에 \(-\infty\)를 넣으면 softmax 후 가중치가 정확히 0이 됨
GPT 계열: 디코더 전용 구조, 항상 Causal Mask 사용
BERT: 마스킹 없는 Bidirectional Attention — 양방향으로 문맥을 볼 수 있음
🔑 BERT vs GPT — 구조 비교
BERT (인코더 전용)
  • Bidirectional Attention (마스크 없음)
  • MLM(Masked Language Modeling)으로 사전학습
  • 분류, NER, QA 등 이해 태스크에 강함
  • 텍스트 생성에는 부적합
GPT (디코더 전용)
  • Causal Mask — 왼쪽 토큰만 봄
  • 다음 토큰 예측으로 사전학습
  • 텍스트 생성, 대화, 요약에 강함
  • ChatGPT, GPT-4 모두 이 구조
