효과크기와
결과 해석 심화

"p < 0.05"는 시작일 뿐입니다. 효과가 얼마나 큰지, 어디서 왜 좋아졌는지, 비용 대비 가치가 있는지 — SCI급 논문의 결과 해석 전략.

🧒 초등학생 한 줄 설명

시험 점수가 평균보다 1점 높아서 통계적으로 유의미하다 해도, 1점 차이가 입시에서 의미 있는가? 효과크기는 "얼마나 다른가"를 본다.

  • p < 0.05 but tiny effect: 10만 명 데이터에서 새 약이 혈압을 0.1mmHg 낮춤 → 통계적 유의미, 임상적 무의미
  • 큰 효과, p > 0.05: 샘플 5명에서 큰 차이가 보였지만 p = 0.12 → 데이터가 더 필요한 것
  • AI에서: "정확도 0.3% 향상 (p < 0.001)"은 상용 시스템에서 의미 있을 수도, 없을 수도 있다. Effect size + 비용이 함께 보고돼야 한다

📐효과크기 (Effect Size) — 차이의 크기를 측정한다

p-value는 "차이가 우연이 아닐 확률"을 말합니다. 그러나 샘플이 매우 많으면 아주 작은 차이도 p < 0.05가 됩니다. 효과크기는 "차이가 실제로 얼마나 큰가"를 측정합니다. SCI 논문에서 두 가지를 함께 보고하는 것이 표준입니다.

Cohen's d — 두 집단 평균 차이의 효과크기
\[ d = \frac{\bar{X}_1 - \bar{X}_2}{s_{\text{pooled}}} \quad\text{where}\quad s_{\text{pooled}} = \sqrt{\frac{(n_1-1)s_1^2 + (n_2-1)s_2^2}{n_1+n_2-2}} \]
\(\bar{X}_1, \bar{X}_2\): 두 그룹의 평균 (예: 제안 모델 vs 기준 모델의 정확도)
\(s_{\text{pooled}}\): 합동 표준편차 (두 그룹의 분산을 샘플 크기로 가중 평균)
\(d = 0.2\): 작은 효과 / \(d = 0.5\): 중간 / \(d \geq 0.8\): 큰 효과 (Cohen's 기준)
단위 없음: 표준편차 단위로 표준화했으므로 서로 다른 스케일도 비교 가능

효과크기 해석 기준

Cohen's d효과 크기해석ML 논문 예시
0.0 ~ 0.2매우 작음실질적 의미 재검토정확도 0.1~0.5% 향상
0.2 ~ 0.5작음주목할 만하나 제한적정확도 1~2% 향상
0.5 ~ 0.8중간실질적으로 의미 있음정확도 3~5% 향상
0.8 이상큼강한 실질적 효과정확도 10% 이상 향상
⚠️

p-value와 effect size를 모두 보고한다. "p = 0.03, d = 0.15"는 통계적으로 유의미하지만 효과가 매우 작다는 뜻이다. "p = 0.08, d = 0.72"는 샘플이 부족해서 유의미성은 낮지만 효과는 크다는 뜻이다. 두 정보를 함께 봐야 한다.

🔬세부 케이스 분석 — 평균이 가리는 진실

전체 평균 정확도가 좋아도, 특정 클래스나 서브그룹에서는 나빠졌을 수 있습니다. SCI급 논문은 평균 외에 세부 분석을 요구합니다. 이 분석이 논문의 Discussion과 Limitation을 풍부하게 만듭니다.

클래스 불균형에서의 F1 비교
\[ \text{Macro-F1} = \frac{1}{K}\sum_{k=1}^K F1_k \qquad\text{vs}\qquad \text{Weighted-F1} = \sum_{k=1}^K \frac{n_k}{N} F1_k \]
\(K\): 클래스 수, \(n_k\): k번째 클래스의 샘플 수, \(N\): 전체 샘플 수
Macro-F1: 모든 클래스를 동등하게 취급 → 소수 클래스 성능 중시
Weighted-F1: 샘플 수에 비례하여 가중 → 다수 클래스 성능 중시
클래스 불균형이 있으면 두 가지 모두 보고해야 함
✅

실패 사례를 논문에 공개하는 것은 강점이다. "Figure 3: 대표적인 실패 사례"처럼 오답 예시를 보여주면, 리뷰어가 "이 저자는 자기 모델의 한계를 정확히 파악하고 있다"고 판단한다. 실패 사례 분석이 Limitation + 향후 연구로 자연스럽게 이어진다.

⚖비용-성능 트레이드오프

SCI급 논문에서 "성능이 1% 올랐다"는 충분하지 않을 수 있습니다. 그 1%를 위해 계산 비용이 10배 늘었다면, 실용적 가치가 있는지 의문입니다. 비용-성능 트레이드오프 분석은 레벨 3 논문의 필수 항목입니다.

🌐일반화 가능성 — 내 결과는 어디까지 통하는가

연구의 일반화 가능성(generalizability)은 결과의 가장 중요한 측면 중 하나입니다. 내 방법이 특정 데이터셋·언어·도메인에서만 좋은지, 아니면 더 넓은 범위에서도 통하는지 논의해야 합니다.

검증 유형방법논문에서의 의미
다중 데이터셋3개 이상 다른 데이터셋에서 실험"다양한 도메인에서 일관된 성능"
Distribution shift학습 분포와 다른 데이터로 테스트"OOD 강건성"
크기 변화더 큰/작은 모델로도 작동하는지"Scaling에 강건한 방법"
언어/도메인 변환영어 모델을 한국어에 적용"언어 독립적 방법"
ℹ️

일반화 가능성의 한계를 솔직하게 적는 것이 강점이다. "본 연구는 의료 영상 도메인에서만 검증됐으며, 자연어 도메인으로의 직접 적용은 추가 실험이 필요하다"처럼 범위를 명확히 하면, 리뷰어가 "범위 밖의 결과를 주장한다"고 공격할 수 없다.

🧠 확인 문제

두 모델을 비교한 논문에서 "p = 0.001, Cohen's d = 0.18"이라면 어떻게 해석해야 하는가?

(a)   p < 0.05이므로 완전히 유의미한 결과다
(b)   Cohen's d < 0.2이므로 이 실험은 실패다
(c)   p-value와 Cohen's d는 같은 정보를 다르게 표현한 것이다
(d)   통계적으로 유의미하지만 효과 크기가 매우 작아 실질적 의미가 제한적이다
