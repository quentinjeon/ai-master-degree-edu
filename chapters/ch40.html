Ch.40 · 연구 방법론 (레벨 3 — Phase 3-C · 14주차 · 최종)

리뷰 대응·Response Letter·재투고 전략

"논문은 초고를 써서 끝나는 일이 아니다." Reject를 accept로 만드는 리뷰 분석, Response letter 작성 구조, 재투고 전략까지 — 논문 출판의 마지막 관문.

🧒 초등학생 한 줄 설명

논문 리뷰어는 적이 아니라 코드 리뷰어다 — PR을 올리면 동료가 "이 부분 다시 보자"고 코멘트 다는 것처럼, 리뷰어도 논문을 더 좋게 만들려고 지적하는 것이다.

  • NeurIPS 2023 accept rate 26% — 세계 최고 연구자도 reject를 받는다. Reject는 실패가 아니라 과정이다
  • 대부분의 논문은 1~3번 수정 후 accept된다. 첫 번째 제출이 마지막이 되는 경우는 드물다
  • AI 연구에서: Response letter를 체계적으로 쓰면 accept 확률이 크게 상승한다 — 리뷰 코멘트에 "Type A~D" 분류를 적용하고 5단계 구조로 답변하는 것이 이 챕터의 핵심이다

🔍리뷰 코멘트 분석 — 유형 분류가 먼저다

리뷰 코멘트를 받으면 즉시 감정적으로 반응하지 말고 먼저 유형을 분류해야 한다. 유형에 따라 대응 방법이 완전히 달라지기 때문이다. 모든 코멘트는 아래 4가지 유형 중 하나로 분류된다.

리뷰 코멘트 4가지 유형 분류
\[ \text{Type A (사실 오류)} \xrightarrow{\text{글쓰기로 해결}} \text{논문 표현 수정} \] \[ \text{Type B (오해)} \xrightarrow{\text{문장 보강}} \text{설명 추가·명확화} \] \[ \text{Type C (실험 부족)} \xrightarrow{\text{추가 실험}} \text{Ablation·Baseline 보완} \] \[ \text{Type D (본질 한계)} \xrightarrow{\text{Limitation 섹션}} \text{미래 연구 방향 명시} \]
Type A: 리뷰어가 논문의 수식·수치를 잘못 읽은 경우 — 원문을 더 명확하게 수정
Type B: 리뷰어가 방법론을 오해한 경우 — 추가 설명·예시·그림으로 명확화
Type C: 리뷰어가 추가 실험을 요구하는 경우 — 수행 가능하면 실험, 불가능하면 이유 설명
Type D: 리뷰어가 연구 자체의 한계를 지적하는 경우 — Limitation 섹션에 솔직하게 기술하고 미래 연구로 연결

흔한 리뷰 코멘트와 분류·대응 방법

리뷰 코멘트 예시유형분류 근거대응 방법
"The baseline comparison is insufficient" Type C 추가 실험 요구 최신 SOTA 2~3개 추가 비교 실험 수행
"The related work section misses [논문명]" Type A 정보 누락 (글쓰기) 해당 논문 인용 추가 + 간단한 비교 서술
"It's unclear how the hyperparameter λ was chosen" Type B 방법론 설명 오해 실험 세팅 섹션에 Hyperparameter search 과정 명시
"The method only works on vision tasks" Type D 본질적 적용 범위 한계 Limitation에 솔직히 기술, NLP 적용 가능성을 Future Work로
"The computational cost is too high for practical use" Type D 본질적 효율 한계 FLOPs 비교 표 추가 + 경량화 방향 Future Work 언급
"Figure 3 is confusing" Type B 표현 불명확 (오해 유발) 그림 재제작 + 캡션 확장, 본문 참조 문장 보강
✅

분류 우선 규칙: 리뷰를 받으면 24시간 이내에 모든 코멘트에 Type A/B/C/D를 붙이는 작업만 먼저 한다. 이 분류가 끝나야 비로소 "얼마나 많은 추가 실험이 필요한가"를 현실적으로 파악할 수 있다.

📝Response Letter 구조 — 5단계 작성법

Response letter는 단순한 답변이 아니라 리뷰어를 설득하는 구조화된 문서다. 아래 5단계 구조를 모든 코멘트에 일관되게 적용하면 리뷰어가 "저자가 코멘트를 진지하게 받아들였다"는 인상을 받는다.

Response Letter 5단계 구조
\[ \text{Step 1: } \underbrace{\text{요약 + 감사}}_{\text{Opening}} \to \text{Step 2: } \underbrace{\text{대응 방향}}_{\text{Action}} \to \text{Step 3: } \underbrace{\text{수정 내용}}_{\text{Change}} \to \text{Step 4: } \underbrace{\text{논문 위치}}_{\text{Location}} \to \text{Step 5: } \underbrace{\text{추가 설명·반박}}_{\text{Discussion}} \]
Step 1: 코멘트를 한 줄로 요약하고, 지적에 감사함을 표현 ("Thank you for this insightful comment...")
Step 2: 어떤 방향으로 대응할지 먼저 밝힘 ("We agree / We partially agree / We respectfully disagree...")
Step 3: 실제로 논문에서 무엇을 변경했는지 기술 (신규 실험, 텍스트 수정, 그림 교체 등)
Step 4: 수정 내용이 논문의 어디에 있는지 페이지·섹션·줄 번호를 명시
Step 5: 리뷰어가 여전히 궁금할 수 있는 사항에 추가 설명. 동의하지 않는 경우 정중하게 반박
1

요약 + 감사 (Opening): 리뷰어의 코멘트를 한 문장으로 압축 인용 후 감사 표현. "We thank Reviewer #2 for pointing out..."처럼 시작.

2

대응 방향 (Action): "We agree", "We partially agree", "We respectfully disagree" 중 하나로 입장을 먼저 밝힌다. 모호한 표현 금지.

3

수정 내용 (Change): 실제 변경 사항을 구체적으로 서술. "We added Table 3 comparing our method with GPT-4 on..."처럼 구체적으로.

4

논문 위치 (Location): "Please see Section 4.2, lines 312–328 of the revised manuscript." — 리뷰어가 직접 확인할 수 있도록 위치 명시.

5

추가 설명·반박 (Discussion): Type C 실험 결과 요약, Type D 한계 인정, 또는 Type A/B 반박 근거 제시. 길어도 괜찮다.

나쁜 표현 vs 좋은 표현 — 실전 변환 예시

나쁜 표현 ❌좋은 표현 ✅이유
"The reviewer misunderstood our method." "We apologize for the unclear presentation. We have revised Section 3.1 to clarify..." 리뷰어를 탓하면 적대감을 유발. 자신의 글쓰기를 개선한다고 표현
"This is beyond the scope of this paper." "We agree this is an important direction. We have added this to the Limitation section (§6) and plan to address it in future work." 단순 거절은 리뷰어를 실망시킴. 인정 + 미래 연구 연결로 전환
"We already compared with all relevant baselines." "Thank you for suggesting [Method X]. We have added a comparison in Table 2 (§4.3). Our method outperforms it by 2.3% on..." 방어적 태도 대신 실험으로 직접 보여줌
"The reviewer is wrong about the complexity." "We respectfully disagree. The complexity is O(n log n), not O(n²), as we show in Appendix A with detailed derivation." "respectfully disagree" + 근거 명시로 정중하되 명확하게 반박
📌

Response letter 전체 구성: 도입부에서 "We thank all reviewers for constructive feedback..." 로 시작. 각 리뷰어 별로 섹션 구분. 코멘트는 그대로 인용(인용 블록)하고 바로 아래 답변. 마지막에 모든 변경 사항을 요약한 "Summary of Changes" 섹션 추가.

🔄Reject 처리와 재투고 전략

Reject는 해당 제출의 끝이지 연구의 끝이 아니다. Reject 메일을 받으면 최소 48시간 동안 논문을 열지 말고 감정을 가라앉힌다. 그 후 리뷰를 냉정하게 분석해 재투고 venue를 결정한다.

재투고 결정 트리 (Decision Tree)
\[ \text{Reject 분석} \begin{cases} \text{비판이 타당 + 수정 가능} & \Rightarrow \text{같은 급 venue 재투고 (수정 후)} \\ \text{오해·misunderstanding 多} & \Rightarrow \text{같은 venue 다음 사이클 재투고} \\ \text{Scope 미스매치} & \Rightarrow \text{더 적합한 venue 변경} \\ \text{근본 결함 발견} & \Rightarrow \text{6개월 추가 작업 후 재투고} \end{cases} \]
"비판이 타당 + 수정 가능": 리뷰어 지적이 맞고, 1~2달 안에 해결 가능한 경우 — 최대한 빠르게 수정해 다음 deadlie 제출
"오해 多": 리뷰어가 방법론을 잘못 이해한 경우 — 같은 venue의 다음 사이클(보통 1년 후)에 설명을 개선해 재제출
"Scope 미스매치": 방법론이 해당 venue 관심사와 맞지 않는 경우 — 더 적합한 workshop·journal 탐색
"근본 결함": 연구 설계 자체의 문제 — 재설계 필요, 급하게 재투고하면 또 reject
비판 타당 + 수정 가능
→
같은 급 재투고: 리뷰를 전부 반영해 수정 후 다음 deadline 제출. NeurIPS reject → ICLR 제출처럼.
오해·불명확 多
→
같은 venue 재투고: 설명을 대폭 개선해 다음 사이클에 같은 학회 재제출. 방법론이 좋으면 통과 확률 높음.
Scope 미스매치
→
Venue 변경: 연구 내용에 더 맞는 학회·저널 탐색. CVPR reject → ECCV, 또는 관련 workshop 제출.
근본 결함 발견
→
6개월 추가 작업: 연구 설계·실험 재설계. 급하게 재투고하면 리뷰어에게 같은 reject를 받는다.
⚠️

Reject 후 즉시 재투고의 위험성: Reject를 받자마자 리뷰를 무시하고 다른 학회에 그대로 제출하면, 같은 리뷰어가 배정될 수 있다 (연구 커뮤니티는 좁다). 또한 수정하지 않은 논문은 같은 약점을 가지므로 또 reject된다. 최소 4주~8주 동안 리뷰를 깊이 반영한 후 재투고한다.

🧠멘탈 관리 — 논문 출판은 마라톤이다

논문 출판 과정에서 심리적 소진(burnout)은 매우 흔하다. 타임라인을 현실적으로 이해하고 각 단계의 심리적 위험을 미리 알면 훨씬 잘 버틸 수 있다.

논문 출판 전체 타임라인
\[ \underbrace{2 \sim 6\text{개월}}_{\text{초고 작성}} + \underbrace{2 \sim 4\text{개월}}_{\text{심사 대기}} + \underbrace{1 \sim 3\text{개월}}_{\text{수정 대응}} + \underbrace{1 \sim 6\text{개월}}_{\text{출판 처리}} = \underbrace{1 \sim 2\text{년}}_{\text{총 소요 시간}} \]
초고 작성: 연구 설계부터 완성된 초고까지 — 실험 실패·방향 전환 포함하면 더 길어질 수 있음
심사 대기: 제출 후 리뷰 결과까지 — 학회 2~4개월, 저널 3~12개월
수정 대응: Minor revision 1~2개월, Major revision 2~3개월
출판 처리: Accept 후 proof, DOI 발급, 학회 발표 준비까지
총 소요: 좋은 논문도 1년 이상 걸리는 것이 정상이다

6단계별 심리적 위험과 대응 전략

단계심리적 위험대응 전략
1. 초고 작성 중 완벽주의로 인한 마비 ("아직 준비가 안 됐어") 초고는 나쁜 것이 정상. 일단 쓰고 수정한다
2. 제출 직후 기대 과잉 + 불안 ("이번엔 accept될까?") 다음 프로젝트를 즉시 시작. 결과를 기다리며 멈추지 않는다
3. Reject 수신 직후 분노, 자기 의심, 포기 충동 48시간 논문 열지 않기. 산책·운동. 리뷰어에게 답장 금지
4. 리뷰 분석 단계 리뷰어 비난, 방어적 해석 모든 코멘트에 Type A~D 분류 작업만 먼저. 감정 판단 뒤로
5. 수정 작업 중 추가 실험 범위 확장으로 인한 소진 Type C 실험 중 "핵심 요청" vs "nice-to-have" 구분 후 우선순위 설정
6. 재심사 대기 다시 불안 증가, 프로젝트 방치 Response letter 완성 → 제출 → 즉시 다음 프로젝트. 논문은 시스템이다
🏆

최고 연구자도 reject를 받는다: Yann LeCun, Geoffrey Hinton, Yoshua Bengio — 이 세 명의 논문도 수차례 reject를 받았다. 논문의 가치와 accept 여부는 다르다. 리뷰를 통해 논문이 더 좋아지고, 결국 출판된다. Reject를 받는다는 것은 충분히 도전적인 venue에 제출했다는 증거다.

🧠 확인 문제

리뷰어가 "Baseline comparison is insufficient"이라는 코멘트를 남겼다. 첫 번째 대응으로 가장 적절한 것은?

(a)   "We already compared with all relevant baselines in Table 1" 이라고 반박한다
(b)   Type C(실험 부족)로 분류하고, 추가 baseline 실험 계획을 수립한다
(c)   핵심 연구 범위를 벗어나므로 해당 코멘트를 무시한다
(d)   "This comparison is beyond the scope of this paper"라고 답변한다
