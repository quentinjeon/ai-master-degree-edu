🔤BPE 토크나이저 — 텍스트를 숫자로 바꾸는 첫 단계

모든 언어 모델은 텍스트를 직접 처리하지 않습니다. 먼저 텍스트를 정수(token ID)의 시퀀스로 변환해야 합니다. BPE(Byte Pair Encoding, Sennrich et al., 2016)는 자주 등장하는 문자 쌍을 반복적으로 합쳐 어휘집(vocabulary)을 만드는 알고리즘입니다.

GPT-4는 약 10만 개, LLaMA 3는 약 12만 8천 개의 어휘집을 사용합니다. 어휘집이 클수록 문장을 더 짧은 토큰 수로 표현할 수 있어 긴 문맥을 처리하는 데 유리합니다.

1

초기화: 모든 개별 바이트(256개)를 어휘집에 추가. 텍스트를 바이트 단위로 분리한다.

2

빈도 계산: 현재 토큰 시퀀스에서 인접한 토큰 쌍의 등장 빈도를 모두 계산한다.

3

합치기(Merge): 가장 자주 나오는 쌍 (a, b)를 새 토큰 ab로 합쳐 어휘집에 추가하고 텍스트를 업데이트.

4

반복: 목표 어휘 크기(V)에 도달할 때까지 2~3을 반복. 최종 어휘 크기 = 256 + V.

BPE MERGE 규칙 (Sennrich et al., 2016)
\[ \text{merge}(a,\,b) \quad \text{s.t.} \quad \text{freq}(ab) = \max_{\,(x,\,y)} \text{freq}(xy) \]
\(\text{freq}(ab)\): 현재 토큰 시퀀스에서 인접 쌍 (a, b)의 등장 횟수
\(\max_{(x,y)}\): 모든 인접 쌍 중 빈도가 최대인 쌍을 선택
merge 1회 → 어휘 크기 +1, V번 반복 → 최종 어휘 크기 = 256 + V
GPT-4 tiktoken: V ≈ 100,000 → 어휘 크기 약 100,256개
🔑 토크나이저 종류 비교
종류대표 모델합치기 기준특징
BPEGPT, LLaMA등장 빈도 최대언어 무관, 범용적
WordPieceBERT우도 증가 최대서브워드 분리에 특화
SentencePieceT5, GemmaBPE + WordPiece 통합공백 포함 raw text 처리
TiktokenGPT-4, o1, o3BPE 기반Rust 구현, 매우 빠름
✅

한국어 BPE 특이점: 영어는 단어 경계가 공백으로 명확하지만 한국어는 형태소(어간+어미)가 복잡합니다. "안녕하세요"는 BPE로 ["안녕", "하세요"]처럼 분리될 수 있고, 이 분리 패턴이 모델의 언어 이해에 직접 영향을 줍니다. 한국어 특화 모델들은 더 큰 어휘집을 사용하는 경향이 있습니다.

⚡Transformer FLOPs 계산 — "학습에 얼마나 걸리는가"

"7B 파라미터 모델을 1조 토큰으로 학습하면 H100 GPU 몇 개가 몇 일 필요한가?" 이 질문에 정확하게 답하려면 FLOPs(Floating Point Operations) 계산이 필수입니다. CS336 L2에서 Percy Liang이 강조하는 Arithmetic Intensity 분석의 핵심이기도 합니다.

TRANSFORMER 학습 총 FLOPS 근사 (Kaplan et al., 2020)
\[ C \approx 6ND \quad\text{where}\quad N = \text{파라미터 수},\; D = \text{학습 토큰 수} \]
\(C\): 총 연산량 (FLOPs = Floating Point Operations)
\(N\): 모델 파라미터 수 (예: LLaMA 3 8B → N = 8×10⁹)
\(D\): 학습에 사용한 토큰 수 (예: 1조 토큰 → D = 10¹²)
계수 6의 유래: 순전파 2×ND + 역전파 4×ND ≈ 6ND
예시: GPT-3 (175B 파라미터, 300B 토큰) → C ≈ 3.1×10²³ FLOPs
TRANSFORMER 레이어당 FLOPS (정밀 계산, CS336 L2)
\[ \text{FLOPs/token/layer} \approx \underbrace{8\,d_{\text{model}}^2}_{\text{Attention (QKV+O)}} + \underbrace{16\,d_{\text{model}}^2}_{\text{FFN (4× 확장)}} = 24\,d_{\text{model}}^2 \]
\(d_{\text{model}}\): 모델 숨겨진 차원 (LLaMA 3 8B: 4096)
Attention: Q·K·V 투영 (3×2d²) + 출력 투영 (2d²) = 8d²
FFN: 내부 차원 = 4×d_model → 게이트·업·다운 투영 = 16d²
L개 레이어 전체: ≈ 24Ld² × 시퀀스 길이 (어텐션 항 별도 추가)
Arithmetic Intensity = FLOPs ÷ HBM 접근 bytes → 높을수록 GPU 효율↑

주요 LLM 학습 비용 비교

모델파라미터 N학습 토큰 D총 FLOPs CGPU 추정

🚀FlashAttention — I/O 최적화의 혁명

표준 Self-Attention은 \(N \times N\) 크기의 어텐션 행렬을 HBM(GPU DRAM)에 기록하고 다시 읽어옵니다. 시퀀스 길이 N = 4096이면 이 행렬 크기가 수백 MB에 달해 메모리 접근이 병목이 됩니다. FlashAttention(Dao et al., 2022)은 이 문제를 SRAM 내부 타일링으로 해결합니다.

표준 Attention — 느림

① Q, K, V를 HBM에서 읽기

② S = QKᵀ 계산 → HBM에 기록

③ P = softmax(S) → HBM에 기록

④ O = PV 계산 → HBM에 기록

HBM 접근: O(N²d) — 느림

FlashAttention — 빠름

① Q, K, V를 SRAM 타일 단위로 로드

② 타일 내에서 S, P, O를 SRAM에서만 계산

③ 다음 타일로 이동 (누적값 유지)

④ 최종 O만 HBM에 기록

HBM 접근: O(N²d/M) — 최대 10× 빠름

FLASHATTENTION I/O 복잡도 (Dao et al., 2022)
\[ \text{표준 HBM 접근량:} \; O(N^2 d) \quad\xrightarrow{\text{FlashAttention}}\quad O\!\left(\frac{N^2 d}{M}\right) \]
\(N\): 시퀀스 길이, \(d\): 어텐션 헤드 차원
\(M\): SRAM 크기 (A100: 192KB/SM, 전체 ~30MB)
속도 향상: \(M/d\)배 — d=64이면 최대 수백× I/O 절감
실제 속도: 2~4× 빠름 (다른 연산도 있어서)
FlashAttention2 (2023): 더 효율적인 타일링 → GPU 활용률 50%→70%↑
💡

Arithmetic Intensity (산술 강도): FLOPs ÷ HBM 접근 bytes. 높을수록 "컴퓨트 바운드" (GPU가 쉬지 않고 계산). MatMul은 ~200 FLOP/byte (컴퓨트 바운드), 표준 Attention은 ~3 FLOP/byte (메모리 바운드). FlashAttention은 타일링으로 Attention을 컴퓨트 바운드에 가깝게 만들어 GPU 효율을 극적으로 높입니다. CS336 L2의 핵심 개념.

📈Scaling Laws — 얼마나 키워야 최적인가

모델을 키우면 성능이 좋아지는 것은 알지만, 파라미터를 키워야 하는가 데이터를 늘려야 하는가? 이 질문에 수학적으로 답한 것이 Scaling Laws입니다. Kaplan et al.(2020)의 초기 연구에서 Hoffmann et al.(2022) Chinchilla 논문까지 이어지는 핵심 통찰입니다.

신경 언어 모델 손실 스케일링 (Kaplan et al., 2020)
\[ L(N,\, D) = \left(\frac{N_c}{N}\right)^{\!\alpha_N} + \left(\frac{D_c}{D}\right)^{\!\alpha_D} + L_\infty \]
\(L(N,D)\): 파라미터 N, 학습 토큰 D에서의 검증 손실 (낮을수록 좋음)
\(\alpha_N \approx 0.076,\; \alpha_D \approx 0.095\): 멱법칙 지수 (데이터 스케일이 약간 더 중요)
\(L_\infty\): 모델·데이터가 무한히 커도 없어지지 않는 하한 (데이터의 엔트로피)
\(N_c,\, D_c\): 스케일링 상수 (데이터로 적합)
해석: N 또는 D를 10배 늘리면 손실이 멱법칙으로 감소
CHINCHILLA 최적 스케일링 (Hoffmann et al., 2022)
\[ N^* = \left(\frac{C}{6a}\right)^{0.5},\quad D^* = \left(\frac{C}{6b}\right)^{0.5} \quad\Rightarrow\quad D^* \approx 20 \cdot N^* \]
\(C = 6ND\): 컴퓨트 예산 (총 FLOPs)
\(a,\, b\): 스케일링 계수 (Hoffmann et al. 논문에서 추정)
핵심 결론: 컴퓨트 고정 시 최적 토큰 수 ≈ 파라미터 수 × 20
예시: 7B 파라미터 → 최적 학습 토큰 ≈ 140B (기존 GPT-3는 이보다 훨씬 적음)
LLaMA 3 8B: 15T 토큰 학습 → Chinchilla 비율(160B)보다 훨씬 많음 → 추론 효율 최대화
🔑 스케일링 법칙의 실전 의미

Chinchilla 이후 AI 업계의 학습 전략이 바뀌었습니다. 이전에는 "같은 컴퓨트면 모델을 크게"가 정설이었지만, 이제는 "추론 비용을 고려하면 작은 모델을 더 오래 학습"하는 것이 합리적입니다. LLaMA 3, Mistral, Gemma가 Chinchilla 비율보다 훨씬 많은 데이터를 학습하는 이유가 바로 이것입니다 — 학습은 한 번이지만 추론은 수억 번 일어납니다.

⚠️

Chinchilla 비율은 "학습 최적"이지 "최종 모델 최적"이 아닙니다: 실제 배포 모델은 추론 속도가 중요하므로 작은 모델 + 더 많은 데이터 전략을 씁니다. 또한 Chinchilla 지수 자체도 데이터 품질·구성에 따라 달라집니다.

🔀병렬화 — 수천 개 GPU를 어떻게 쓰는가

LLM 학습에는 수천 개의 GPU가 필요합니다. 이를 효율적으로 사용하려면 세 가지 병렬화 전략을 조합합니다. CS336 L7~L8에서 Tatsu Hashimoto가 상세히 설명합니다.

병렬화 종류분할 대상특징한계
데이터 병렬 (DP)미니배치구현 단순, 선형 스케일모델이 GPU 1개에 들어가야 함
텐서 병렬 (TP)행렬 연산 내부대형 모델 가능통신 오버헤드 큼, 같은 노드 권장
파이프라인 병렬 (PP)레이어 단위노드 간 통신 최소버블(유휴 시간) 발생
MFU — 모델 FLOPs 활용률 (CS336 L2)
\[ \text{MFU} = \frac{\text{실제 달성 FLOPs/s}}{\text{이론 최대 FLOPs/s}} \quad\Rightarrow\quad \text{실효 연산} = \text{GPU 피크} \times \text{MFU} \]
H100 SXM5 피크: 1979 TFLOP/s (BF16 Tensor Core 기준)
실제 LLM 학습 MFU: 보통 35~50% (네트워크·메모리 병목 등)
MFU 50% 달성이 "효율적인 학습"의 기준점으로 여겨짐
FlashAttention, 최적 배치 크기, 그래디언트 체크포인팅이 MFU에 영향
