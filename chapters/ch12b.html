Ch.12B · LLM 학습 (Unit 3 고도화)

LLM 사전학습·파인튜닝·LoRA

ChatGPT가 어떻게 학습되는지 — 사전학습, 지도 파인튜닝(SFT), 그리고 LoRA로 1% 파라미터만 바꿔 성능을 끌어올리는 비결.

🧒 초등학생 한 줄 설명

"요리사를 키우는 과정과 같다. 먼저 수백만 권의 요리책을 읽혀서 '일반 요리 감각'을 익히고(사전학습), 그다음 특정 식당 스타일에 맞게 짧게 추가 훈련한다(파인튜닝). LoRA는 요리책 전체를 새로 쓰는 대신 '핵심 레시피 메모지'만 추가하는 방법이다."

  • 사전학습: GPT-4는 인터넷 전체 텍스트로 사전학습 → 다음 토큰 예측을 수조 번 반복
  • SFT: "의사처럼 대화하는 방법"을 수천 개 예시로 추가 학습
  • LoRA: 1750억 파라미터 모델을 파인튜닝할 때 단 0.1%만 업데이트 → GPU 비용 1/100
  • AI에서: LLaMA 3, Mistral, Gemma 등이 모두 LoRA로 도메인 특화 — 소비자 GPU로도 대형 모델 파인튜닝 가능

📚사전학습 — 조 단위 토큰으로 언어 구조 흡수

사전학습(Pretraining)은 LLM이 특정 태스크 없이 인터넷 텍스트 전체를 학습하는 단계입니다. 목표는 단순합니다: "다음 토큰을 예측하라." 하지만 이 단순한 목표가 수학, 코딩, 논문 독해 능력을 자연스럽게 만들어냅니다.

사전학습 목적함수 — 자기회귀 NLL (Negative Log-Likelihood)
\[ \mathcal{L}_{\text{PT}}(\theta) = -\sum_{t=1}^{T} \log P_\theta(w_t \mid w_1, \ldots, w_{t-1}) \]
\(w_t\): \(t\)번째 토큰 — 어휘(vocabulary)에서 하나의 단어/서브워드
\(\theta\): LLM의 전체 파라미터 (가중치 행렬들)
\(T\): 시퀀스 전체 길이
모든 위치에서 이전 토큰들을 조건으로 다음 토큰을 맞히도록 경사하강법으로 최적화
사전학습 후 모델 = "언어를 이해하는 만능 기반" (Foundation Model) — 특정 태스크 명세 없이도 범용 능력 획득
🔑 사전학습 스케일의 비밀

사전학습의 성능은 데이터·파라미터·연산량 세 가지 스케일에 따라 결정됩니다:

  • 데이터: 수조(trillion) 토큰 — Common Crawl, Wikipedia, GitHub, arXiv 등 혼합
  • 규모: GPT-3 = 1,750억 파라미터 / LLaMA 3 = 700억 파라미터
  • 비용: GPT-4 사전학습 추정 ~$100M 이상 (A100 GPU 수천 장)
  • Chinchilla 법칙: 최적 비율 — 토큰 수 ≈ 파라미터 수 × 20 (Hoffmann et al., 2022)

🎯지도 파인튜닝 — "이렇게 대화해" 가르치기

SFT(Supervised Fine-Tuning)는 사전학습된 모델에게 "올바른 대화 방식"을 가르치는 단계입니다. (질문, 이상적 대답) 쌍을 수천~수만 개 준비하여 같은 NLL 손실로 학습합니다. 사전학습과 수식 구조는 동일하지만, 응답(y) 부분의 토큰에만 손실을 계산한다는 점이 핵심 차이입니다.

SFT 목적함수 — Supervised Fine-Tuning Loss
\[ \mathcal{L}_{\text{SFT}}(\theta) = -\sum_{(x,y) \in \mathcal{D}_{\text{SFT}}} \sum_{t=1}^{|y|} \log P_\theta(y_t \mid x,\, y_{ \(x\): 사용자 질문(prompt) — 모델에게 주어지는 입력
\(y\): 이상적 응답, \(y_t\)는 \(t\)번째 응답 토큰
\(\mathcal{D}_{\text{SFT}}\): (질문, 응답) 쌍으로 구성된 지도 학습 데이터셋
사전학습과 수식은 동일하지만 응답 \(y\) 부분의 토큰만 손실 계산 — 질문 \(x\) 부분은 손실에서 제외
InstructGPT(ChatGPT 전신)는 약 13,000개 (질문, 응답) 쌍으로 SFT 진행

사전학습 vs SFT 비교

항목 사전학습 SFT
데이터 인터넷 전체 (수조 토큰) 큐레이션된 대화 (수만 쌍)
목적 언어 구조 학습 지시 따르기 (Instruction Following)
비용 매우 높음 ($M ~ $B) 상대적으로 낮음 ($K ~ $M)
효과 범용 능력 부여 행동 방향 설정 (도움·무해함·정직)
손실 계산 대상 모든 위치의 토큰 응답(y) 토큰만
📄

InstructGPT (Ouyang et al., 2022): GPT-3 → ChatGPT의 핵심 레시피. SFT → RLHF 순서로 학습. SFT 단계에서 인간 작성 고품질 응답 13K쌍 사용. 사전학습 대비 100배 적은 데이터로도 사용자 선호도가 크게 향상됨.

⚡LoRA — 1%만 업데이트해도 충분하다

전체 파인튜닝(Full Fine-Tuning)은 700억 파라미터를 모두 업데이트하여 엄청난 GPU 메모리가 필요합니다. LoRA(Low-Rank Adaptation)는 가중치 업데이트 행렬 \(\Delta W\)를 두 개의 작은 행렬 \(B \cdot A\)로 분해하여, 실제로 업데이트할 파라미터를 99% 줄입니다.

LoRA — 저차원 업데이트 분해 (Hu et al., 2022)
\[ W' = W_0 + \Delta W = W_0 + BA \quad\text{where}\quad B \in \mathbb{R}^{d \times r},\; A \in \mathbb{R}^{r \times k},\; r \ll \min(d, k) \]
\(W_0 \in \mathbb{R}^{d \times k}\): 사전학습된 원본 가중치 — 고정(freeze), 업데이트 안 함
\(B,\, A\): 실제로 학습할 작은 행렬. \(B\)는 0으로, \(A\)는 가우시안 랜덤으로 초기화 → 초기 \(\Delta W = 0\)
\(r\): 랭크(rank) — 보통 4~64. \(r\)이 작을수록 파라미터 수가 적음
파라미터 수 비교: 원본 \(d \times k\) → LoRA: \(r \times (d + k)\)
예: \(r=8,\; d=k=4096\)이면 원본 16,777,216 → LoRA 65,536 ≈ 99.6% 절감
LoRA 파라미터 효율 계산 — 절감률 공식
\[ \text{절감률} = 1 - \frac{r(d + k)}{dk} \approx 1 - \frac{r}{\min(d,k)} \]
\(r = 8,\; d = k = 4096\): 절감률 \(\approx 99.6\%\)
\(r = 64,\; d = k = 4096\): 절감률 \(\approx 96.9\%\) — 랭크를 높여도 여전히 큰 절감
실제 LoRA는 Transformer의 Attention \(Q, K, V, O\) 행렬에 선택적으로 적용
QLoRA (Dettmers et al., 2023): INT4 양자화 + LoRA = 소비자 GPU 단 1장으로도 70B 모델 파인튜닝 가능
💡

QLoRA (Dettmers et al., 2023): 4비트 양자화(INT4) + LoRA를 결합. 단일 RTX 3090 GPU (24GB VRAM)로도 LLaMA 70B 파인튜닝이 가능하게 만든 혁신. 기존 FP16 LoRA 대비 메모리 사용량 약 4배 감소. NeurIPS 2023 Outstanding Paper.

🗜양자화 — FP32에서 INT4로, 메모리 1/8

LLM의 가중치는 보통 FP32(32비트 부동소수점)로 저장됩니다. 양자화(Quantization)는 이를 INT8 또는 INT4로 압축하여 메모리와 연산 비용을 줄입니다. LoRA와 결합하면 소비자 GPU에서도 대형 모델을 다룰 수 있게 됩니다.

선형 양자화 (Linear Quantization)
\[ w_{\text{quant}} = \text{round}\!\left(\frac{w - z}{s}\right) \quad\text{역양자화:}\quad \hat{w} = s \cdot w_{\text{quant}} + z \]
\(s\) (scale): 양자화 간격 \(= \dfrac{\max(w) - \min(w)}{2^{\text{bits}} - 1}\) — 실수 범위를 정수 범위로 균일 분할
\(z\) (zero-point): 0이 어떤 정수 값에 매핑되는지 — 비대칭 양자화에서 필요
\(w_{\text{quant}}\): 양자화된 정수 가중치 (INT8이면 −128~127, INT4이면 −8~7)
INT8: FP32 대비 4배 압축 / 정확도 손실 미미
INT4: FP32 대비 8배 압축 / 정확도 약간 손실 — NF4(Normal Float 4) 방식으로 개선 가능
📊 비트폭별 특성 비교
  • FP32: 4 bytes/파라미터, 정밀도 최고
  • FP16/BF16: 2 bytes/파라미터, 훈련 표준
  • INT8: 1 byte/파라미터, 추론에 안전
  • INT4: 0.5 bytes/파라미터, QLoRA 활용
🔗 LoRA + 양자화 조합 효과
  • 사전학습 가중치: INT4 고정 (메모리 8배 절감)
  • LoRA 어댑터: FP16으로 학습 (정밀도 유지)
  • 70B 모델 기준: ~140GB → ~35GB로 감소
  • 단일 A100 80GB 또는 RTX 3090×2로 가능
