Ch.02D · 최적화 실습 (Phase 1 보강 · 90일 Day 51~60)

경사하강법 직접 구현
\(\theta = \theta - \eta\nabla J(\theta)\)를 numpy로

수식이 코드가 됩니다. 경사하강법을 scratch로 구현하면 모든 ML 논문의 Algorithm 섹션이 읽힙니다.

🧒 초등학생 한 줄 설명

산에서 눈 감고 내려오기 — "발아래 경사가 가파르면 크게 이동, 평탄하면 조금 이동"

  • 경사(gradient) = 지금 서 있는 곳의 기울기. 이 방향이 "가장 빠르게 올라가는 방향"이다
  • 학습률(η) = 한 걸음의 크기. 너무 크면 산을 건너뛰어 버리고, 너무 작으면 며칠이 걸린다
  • 반복 = 한 걸음씩 계속 내려가기. 반복이 쌓이면 결국 가장 낮은 곳(손실 최솟값)에 도달한다
  • AI에서: 모든 딥러닝 학습의 핵심이 이것. "Adam", "SGD", "RMSProp" 등 다 이 원리의 변형이다

🔄\(\theta = \theta - \eta\nabla J(\theta)\) 를 한 줄씩 코드로

논문에서 "Algorithm 1"로 등장하는 경사하강법을 그대로 Python 코드로 옮깁니다. 수식의 각 기호가 코드의 어느 부분에 대응되는지 1:1로 연결합니다.

경사하강법 업데이트 규칙
\[ \theta^{(t+1)} = \theta^{(t)} - \eta \cdot \nabla_\theta J\!\left(\theta^{(t)}\right) \]
\(\theta\): 학습 파라미터 — 선형회귀에서는 \(w\)(가중치)와 \(b\)(편향)
\(\eta\) (eta): 학습률 — 한 번에 얼마나 이동할지 결정하는 하이퍼파라미터
\(\nabla_\theta J(\theta)\): 손실 \(J\)의 \(\theta\)에 대한 gradient (현재 위치의 기울기 벡터)
\(t \to t+1\): 한 번의 업데이트 = 한 스텝. 이를 반복해 최솟값으로 수렴한다

수식 → 코드 1:1 대응 구현

🔑 4단계 루프 — 모든 딥러닝의 뼈대

Forward → Loss → Gradient → Update. 아래 코드의 각 단계가 수식의 어느 부분인지 주석으로 연결했습니다.

# θ = θ - η · ∇J(θ) 그대로 코드로
# 선형회귀: J(w,b) = (1/n)Σ(ŷᵢ - yᵢ)²

import numpy as np

# 합성 데이터 생성 (공부시간 → 점수)
np.random.seed(42)
X = np.random.rand(100, 1) * 10   # 공부시간 0~10시간
y = 5 * X.squeeze() + 50 + np.random.randn(100) * 3  # y = 5x + 50 + 노이즈

# 파라미터 초기화 (θ = [w, b])
w = 0.0   # 가중치 초기값
b = 0.0   # 편향 초기값
lr = 0.01  # η (학습률)
n = len(X)

# 반복 학습 루프
for epoch in range(100):
    # 1. 예측 (Forward) — ŷ = wᵀx + b
    y_pred = w * X.squeeze() + b

    # 2. 손실 계산 (MSE) — J(θ) = (1/n)Σ(ŷ - y)²
    loss = np.mean((y_pred - y) ** 2)

    # 3. Gradient 계산 (∇J(θ))
    grad_w = (2/n) * np.dot(X.squeeze(), (y_pred - y))  # ∂J/∂w
    grad_b = (2/n) * np.sum(y_pred - y)                 # ∂J/∂b

    # 4. 파라미터 업데이트 (θ = θ - η·∇J) ← 이것이 경사하강법
    w = w - lr * grad_w
    b = b - lr * grad_b

    if epoch % 10 == 0:
        print(f"Epoch {epoch:3d} | Loss: {loss:.4f} | w={w:.3f} | b={b:.3f}")

print(f"\n최종: w={w:.3f} (정답: 5.0), b={b:.3f} (정답: 50.0)")
✅

코드의 4단계가 모든 딥러닝의 뼈대다. Forward → Loss → Gradient → Update. PyTorch에서는 model(x) → loss = criterion(...) → loss.backward() → optimizer.step()으로 표현한다. 같은 4단계다. 프레임워크가 바뀌어도 이 구조는 절대 바뀌지 않는다.

⚖️학습률 \(\eta\) — 너무 크면 발산, 너무 작으면 안 수렴

학습률(η)은 경사하강법에서 가장 중요한 하이퍼파라미터입니다. 너무 크면 최솟값을 넘어서 발산하고, 너무 작으면 수렴이 너무 느립니다. 이 값을 "적절히" 설정하는 것이 실제 ML 엔지니어링의 핵심 과제입니다.

학습률에 따른 결과 비교

학습률 η결과이유
0.001느리게 수렴한 걸음이 너무 작다 — 많은 에폭이 필요
0.01✅ 적절히 수렴균형 잡힌 이동 — 실전에서 자주 쓰이는 기본값
0.1빠르게 수렴 또는 불안정데이터·모델에 따라 다름 — 주의 필요
1.0⚠️ 발산 (loss 증가)너무 크게 건너뜀 — 최솟값을 지나쳐버림
# 학습률에 따른 차이 실험
for lr in [0.001, 0.01, 0.1, 1.0]:
    w, b = 0.0, 0.0
    for epoch in range(50):
        y_pred = w * X.squeeze() + b
        loss = np.mean((y_pred - y) ** 2)
        grad_w = (2/n) * np.dot(X.squeeze(), (y_pred - y))
        grad_b = (2/n) * np.sum(y_pred - y)
        w -= lr * grad_w
        b -= lr * grad_b
    final_loss = np.mean((w * X.squeeze() + b - y) ** 2)
    print(f"η={lr}: 최종 loss={final_loss:.2f}")
수렴 조건 (논문 연결)
\[ \text{수렴 조건 (볼록 함수):} \quad \eta < \frac{2}{L} \quad \text{where } L = \text{Lipschitz constant of } \nabla J \]
\(L\): 기울기 변화의 최대 속도 — Lipschitz 상수 (함수가 "얼마나 빠르게 구부러지는가")
\(\eta < 2/L\): 이 조건을 만족하면 수렴이 수학적으로 보장된다
이것이 Phase 2 Ch.15~16에서 배울 수렴 이론의 기초
논문의 "we set learning rate \(\eta = 0.01\)" 뒤에 이 수학이 있다 — 임의의 선택이 아니다

🎲SGD — 전체 데이터 대신 일부만 써서 업데이트

지금까지 배운 것은 "전체 데이터의 gradient"로 업데이트하는 Batch GD입니다. 실제 딥러닝에서는 데이터 일부(mini-batch)만 써서 빠르게 업데이트하는 SGD(Stochastic Gradient Descent)를 씁니다. ImageNet(130만 장)을 한 번에 올릴 수 없기 때문입니다.

GD 종류 비교

종류한 번에 쓰는 데이터장점단점
Batch GD전체 (n개)정확한 gradient느림, 메모리 많이 필요
SGD1개빠름, 노이즈로 local minimum 탈출매우 불안정, 수렴 보장 어려움
Mini-batch GDk개 (32, 64, 128...)균형 — GPU 병렬 연산 효율 최적batch size 튜닝 필요
# Mini-batch GD 구현
batch_size = 32
w, b = 0.0, 0.0
lr = 0.01

for epoch in range(50):
    # 데이터 섞기
    idx = np.random.permutation(n)
    X_shuffle, y_shuffle = X[idx], y[idx]

    # mini-batch로 나눠서 업데이트
    for i in range(0, n, batch_size):
        X_batch = X_shuffle[i:i+batch_size].squeeze()
        y_batch = y_shuffle[i:i+batch_size]

        y_pred = w * X_batch + b
        grad_w = (2/len(X_batch)) * np.dot(X_batch, (y_pred - y_batch))
        grad_b = (2/len(X_batch)) * np.sum(y_pred - y_batch)
        w -= lr * grad_w
        b -= lr * grad_b

    loss = np.mean((w * X.squeeze() + b - y) ** 2)
    if epoch % 10 == 0:
        print(f"Epoch {epoch} | Loss: {loss:.4f}")
📄

논문에서 "We train with batch size 256, learning rate 0.001..."처럼 쓰인다. 이제 이 문장이 "256개씩 mini-batch로 경사하강법을 적용했다"는 뜻임을 안다. 이 챕터를 마치면 논문의 Training Details 섹션을 완전히 읽을 수 있다.

✅ 구현 체크리스트 (90일 Day 51~60)
MSE 손실의 Gradient (편미분)
\[ J(w, b) = \frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i)^2 \quad \Rightarrow \quad \frac{\partial J}{\partial w} = \frac{2}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i)x_i, \quad \frac{\partial J}{\partial b} = \frac{2}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i) \]
\(J(w,b)\): 평균 제곱 오차(MSE) — 예측값과 실제값의 차이를 제곱해서 평균낸 것
\(\partial J / \partial w\): \(w\)에 대한 편미분 — gradient의 \(w\) 성분. 코드에서 grad_w
\(\partial J / \partial b\): \(b\)에 대한 편미분 — gradient의 \(b\) 성분. 코드에서 grad_b
이 두 gradient를 계산하고 \(w, b\)를 각각 업데이트하면 경사하강법 1스텝이 완성된다
⚠️

흔한 실수 — w -= lr * grad_w vs w += lr * grad_w: gradient 방향이 loss가 증가하는 방향이므로 반드시 빼야 합니다. "경사하강"은 gradient의 반대 방향으로 이동하는 것입니다. +=를 쓰면 loss가 증가(gradient ascent)합니다.
