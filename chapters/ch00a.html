Ch.1A · 기초 수학 브릿지 (1~2주차)

로그와 지수함수

AI 논문에서 log가 넘쳐나는 이유를 이해합니다. Cross-Entropy, 로그우도, Softmax — 모두 여기서 출발합니다.

🧒 쉽게 이해하기

로그는 "몇 번 곱해야 하는가?"라는 질문이다. 지수는 "몇 번 곱한 결과인가?"다.

  • 지수: 2를 3번 곱하면 8이다 → \(2^3 = 8\). 세균이 1시간마다 2배로 늘면 10시간 후 \(2^{10} = 1024\)마리.
  • 로그: 8이 되려면 2를 몇 번 곱해야 하나? → \(\log_2 8 = 3\). "자릿수를 세는 것"이다. 100은 자릿수 2, 1000은 자릿수 3.
  • AI 연결: 딥러닝이 log를 쓰는 이유 — 0.001 × 0.002 × 0.003처럼 작은 확률을 여러 개 곱하면 컴퓨터가 0으로 처리해버린다. log를 쓰면 곱셈이 덧셈으로 바뀌어 안전하게 계산된다.

eˣ 지수함수

지수함수는 "일정한 비율로 성장하거나 감소하는" 현상을 표현합니다. 가장 중요한 밑(base)은 자연상수 \(e \approx 2.718\)입니다.

자연상수 e란?

\(e\)는 "1원을 이자율 100%로, 이자를 매우 자주 복리로 계산하면 결국 얼마가 될까?"의 답입니다. 수학적으로는 \(e = \lim_{n\to\infty}(1+\frac{1}{n})^n \approx 2.71828\cdots\)으로 정의됩니다.

지수함수의 핵심 성질
\[ e^{a+b} = e^a \cdot e^b \qquad e^0 = 1 \qquad \frac{d}{dx}e^x = e^x \]
\(e^{a+b} = e^a \cdot e^b\): 지수의 덧셈 = 밑의 곱셈 (로그와 정반대 관계)
\(\frac{d}{dx}e^x = e^x\): 미분해도 자기 자신 — 이것이 \(e\)를 밑으로 쓰는 이유
💡

AI에서 eˣ 등장 위치: Softmax 분모 \(\sum_j e^{z_j}\), Sigmoid \(\sigma(x) = \frac{1}{1+e^{-x}}\), 정규분포 PDF \(\exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)\)

ln 로그함수

로그함수는 지수함수의 역함수입니다. \(y = e^x\)이면 \(x = \ln y\). 자연로그 \(\ln\)은 밑이 \(e\)인 로그입니다. AI 논문에서 "log"는 대부분 자연로그 \(\ln\)을 의미합니다.

로그의 핵심 성질 (암기 필수)
\[ \ln(ab) = \ln a + \ln b \qquad \ln\!\left(\frac{a}{b}\right) = \ln a - \ln b \qquad \ln(a^n) = n \ln a \]
\(\ln(ab) = \ln a + \ln b\): 곱셈 → 덧셈으로 변환 (가장 중요한 성질)
\(\ln(a^n) = n \ln a\): 지수가 앞으로 내려옴 — MLE에서 곱을 합으로 바꿀 때 사용
\(\ln 1 = 0\), \(\ln e = 1\), \(\ln x\)는 \(x>0\)에서만 정의

로그 핵심 값 표

식값의미AI 활용 상황
\(\ln 1\)\(0\)확률 1 (확실) → 손실 0완벽한 예측의 CE loss
\(\ln e\)\(1\)자연로그의 기준값정규화 상수 계산
\(\ln(0^+)\)\(-\infty\)확률 0 → 손실 무한대예측 0은 절대 안 됨!
\(\ln(0.5)\)\(\approx -0.693\)50% 확률의 로그우도이진 분류 기준점
\(\ln(ab)\)\(\ln a + \ln b\)곱 → 합 변환독립 샘플 결합 확률

🔗 AI에서의 로그·지수

로그와 지수는 현대 딥러닝의 거의 모든 핵심 수식에 등장합니다. 아래 세 가지는 반드시 이해해야 합니다.

Cross-Entropy Loss — 분류 문제의 손실함수
\[ \mathcal{L}_{CE} = -\sum_{k=1}^{K} y_k \log \hat{p}_k \]
\(y_k\): 실제 레이블 (정답 클래스만 1, 나머지 0)
\(\hat{p}_k\): 모델이 예측한 k번 클래스 확률 (0~1)
\(-\log(\hat{p}_k)\): \(\hat{p}_k\)가 1에 가까울수록 손실 0, 0에 가까울수록 손실 무한대
직관: 정답 클래스에 높은 확률을 줄수록 손실이 줄어든다
Softmax — 로짓을 확률로 변환
\[ \hat{p}_k = \text{softmax}(z)_k = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}} \]
\(z_k\): k번 클래스의 로짓 (raw score, 범위 제한 없음)
분모 \(\sum_j e^{z_j}\): 모든 클래스 확률의 합이 1이 되도록 정규화
\(e^{z_k}\): 항상 양수 → 확률로 쓸 수 있음. 지수함수가 필수인 이유
Log-Likelihood — MLE의 목적함수
\[ \log \mathcal{L}(\theta) = \sum_{i=1}^{n} \log p(x_i \mid \theta) \quad \leftarrow \quad \prod_{i=1}^{n} p(x_i \mid \theta) \text{ 의 로그} \]
\(\prod p(x_i|\theta)\): 독립 샘플의 결합 확률 (매우 작은 수들의 곱 → underflow 위험)
\(\sum \log p(x_i|\theta)\): 로그 변환으로 곱 → 합, 수치 안정성 확보
\(\ln(a \cdot b) = \ln a + \ln b\) 성질 덕분에 가능
⚠️

흔한 오해: \(\log\)와 \(\ln\)은 다르다? — AI 논문에서 밑을 명시하지 않은 \(\log\)는 관례상 자연로그(\(\ln\))입니다. 정보이론에서는 \(\log_2\)를 쓰기도 하지만, 결과의 단위(bit vs nat)만 달라질 뿐 같은 개념입니다.

🧠 확인 문제

딥러닝 학습에서 \(\prod_{i=1}^{n} p(x_i|\theta)\)를 직접 계산하지 않고 \(\sum \log p(x_i|\theta)\)를 최대화하는 가장 중요한 이유는?

(a)   log가 더 크게 수렴하므로 최적화에 유리하다
(b)   log를 쓰면 정답률이 높아진다
(c)   log는 항상 양수이므로 확률처럼 쓸 수 있다
(d)   작은 확률들의 곱이 0으로 수렴하는 수치 불안정(underflow)을 방지하고, 곱을 합으로 변환해 계산을 단순하게 만든다
