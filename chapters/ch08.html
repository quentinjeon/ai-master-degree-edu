MAP, Bias-Variance Tradeoff, 가설검정

너무 단순한 모델도, 너무 복잡한 모델도 위험합니다. 좋은 추정이 무엇인지 배웁니다.

🧒 초등학생 한 줄 설명

너무 단순한 모델은 패턴을 못 보고(underfitting), 너무 복잡한 모델은 노이즈까지 외운다(overfitting). — "딱 적당한 복잡도를 찾는 것이 Bias-Variance Tradeoff다."

  • 과소적합: 시험을 준비 못 해서 모든 문제에 "2번"이라고 쓰는 것 (너무 단순)
  • 과적합: 작년 시험 문제를 통째로 외워서 올해 새 문제는 못 푸는 것 (너무 복잡)
  • AI에서: 적절한 모델 크기, 정규화, 드롭아웃 → 좋은 일반화 (generalization)

🎯MAP 추정 (MLE + Prior)

MAP(Maximum A Posteriori)는 MLE에 사전 지식(prior)을 추가한 추정법입니다. MLE가 데이터만 보고 판단하는 반면, MAP는 데이터와 우리가 미리 가진 지식(prior)을 함께 고려합니다.

MAP VS MLE
\[ \hat{\theta}_{MLE} = \operatorname{argmax}_\theta \log P(\mathcal{D} \mid \theta) \quad \text{vs} \quad \hat{\theta}_{MAP} = \operatorname{argmax}_\theta \left[ \log P(\mathcal{D} \mid \theta) + \log P(\theta) \right] \]
\(\hat{\theta}_{MLE}\): MLE 추정값 — 데이터만 봄 (데이터가 적으면 과적합 위험)
\(\hat{\theta}_{MAP}\): MAP 추정값 — 데이터 + prior를 함께 봄
\(\log P(\mathcal{D} \mid \theta)\): 로그 우도(log-likelihood) — 데이터가 파라미터 θ 아래서 관측될 확률
\(\log P(\theta)\): 로그 사전확률(log-prior) — 파라미터에 대한 사전 지식 (정규화 역할)
Gaussian prior \(P(\theta)\) → L2 정규화와 수학적으로 동치
🔑 정규화 = MAP의 prior

L2 정규화: loss + λ‖θ‖² = Gaussian prior \(\mathcal{N}(0, 1/\lambda)\) 가정

L1 정규화: loss + λ‖θ‖₁ = Laplace prior 가정

즉, 정규화는 베이지안 해석에서 prior를 추가하는 것과 수학적으로 동일합니다. 정규화 강도 λ를 높이면 prior의 영향이 커져 파라미터가 0에 가까워집니다.

⚖️Bias-Variance Tradeoff

모델의 예측 오류는 bias²과 variance의 합으로 분해됩니다. 이 두 항목은 서로 상충관계(tradeoff)에 있어, 하나를 줄이면 다른 하나가 커지는 경향이 있습니다.

BIAS-VARIANCE 분해
\[ \mathbb{E}\left[(\hat{f}(x) - y)^2\right] = \underbrace{\text{Bias}[\hat{f}(x)]^2}_{\text{과소적합}} + \underbrace{\text{Var}[\hat{f}(x)]}_{\text{과적합}} + \underbrace{\sigma^2_\epsilon}_{\text{줄일 수 없는 노이즈}} \]
\(\mathbb{E}[(\hat{f}(x) - y)^2]\): 기댓값 예측 오류 (Mean Squared Error)
\(\text{Bias}[\hat{f}(x)]^2\): 모델이 평균적으로 얼마나 틀리는지 — 너무 단순한 모델 → 큰 bias (과소적합)
\(\text{Var}[\hat{f}(x)]\): 훈련 데이터에 따라 예측이 얼마나 흔들리는지 — 너무 복잡한 모델 → 큰 variance (과적합)
\(\sigma^2_\epsilon\): 데이터 자체의 노이즈 — 어떤 모델도 줄일 수 없음 (irreducible error)

Bias vs Variance 비교

Bias 큼 Variance 큼
모델 복잡도 너무 단순 너무 복잡
상태 과소적합 (underfitting) 과적합 (overfitting)
해결책 모델 복잡도 증가, 특성 추가 정규화, 드롭아웃, 더 많은 데이터
예시 선형 모델로 비선형 패턴 학습 100차 다항식으로 10개 데이터 학습

🔬통계적 가설 검정

논문에서 "이 방법이 정말 더 좋은가?"를 통계적으로 검증하는 방법입니다. 가설검정은 데이터를 근거로 주장의 타당성을 판단하는 엄밀한 절차입니다.

1

귀무가설 H₀ 설정: "두 방법 사이에 차이가 없다" — 기본 가정으로 기각하려는 가설

2

대립가설 H₁ 설정: "새로운 방법이 더 좋다" — 증명하려는 주장

3

검정 통계량 계산: 데이터로부터 t-통계량, z-통계량 등 계산하여 수치화

4

p-value 계산: p < α (보통 0.05)이면 귀무가설 기각 → 대립가설 채택

🔑 p-value의 정확한 의미

p-value = "귀무가설이 사실이라고 가정했을 때, 현재 관측된 데이터(또는 더 극단적인 결과)가 나올 확률"

예: p = 0.03 → H₀이 사실이면 이런 데이터가 나올 확률이 3% → H₀ 기각 (통계적으로 유의미한 차이 존재)

⚠️

흔한 오해:

p-value는 "가설이 옳을 확률"이 아닙니다. p < 0.05가 실용적 중요성을 보장하지 않으며(효과 크기, Cohen's d 함께 보기), 표본이 크면 의미 없는 차이도 p < 0.05가 나올 수 있습니다.
