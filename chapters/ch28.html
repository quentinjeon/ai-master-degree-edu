Ch.28 · 통계적 학습 이론 (Phase 3 — 2주차)

PAC 학습 이론 — 얼마나 많은 데이터가 필요한가

Probably Approximately Correct. 표본 복잡도, 일관성, 일반화 오차. 논문의 "sample complexity O(d/ε²)" 분석을 이해합니다.

🧒 초등학생 한 줄 설명

"스팸 필터를 학습시키려면 얼마나 많은 이메일을 봐야 할까?" PAC 이론은 원하는 정확도와 신뢰도를 달성하기 위해 최소 몇 개의 데이터가 필요한지 수학적으로 보장합니다.

  • ε-정확: 학습된 분류기의 진짜 오차가 최적 오차보다 ε 이하 — "거의 맞는(approximately correct)" 조건
  • 1-δ 신뢰도: 이 보장이 1-δ의 확률로 성립 (δ는 실패 확률) — "아마도(probably)" 조건
  • AI에서: 논문에서 "Algorithm X achieves ε-accurate classifier with n = O(d log(1/δ)/ε²) samples"라고 쓰는 것이 PAC 복잡도 결과

🎓PAC 학습 프레임워크

PAC(Probably Approximately Correct) 학습 이론은 학습 문제를 수학적으로 엄밀하게 정의합니다. "언제 어떤 조건에서 학습이 가능한가"를 형식화하며, 학습 알고리즘의 성능 보장을 제공합니다.

PAC LEARNABILITY
\[ \text{가설 클래스 } \mathcal{H} \text{ 가 PAC 학습 가능} \iff \exists\, A,\; m_\mathcal{H}(\varepsilon,\delta):\; \forall D,\; n \geq m_\mathcal{H}(\varepsilon,\delta):\; P\!\left[L_D(A(S)) \leq \min_{h\in\mathcal{H}}L_D(h)+\varepsilon\right] \geq 1-\delta \]
\(\varepsilon\): 정확도 파라미터 — 작을수록 엄격한 정확도 요구
\(\delta\): 신뢰도 파라미터 — 작을수록 실패 확률이 낮아짐
\(L_D(h)\): 분포 \(D\)에 대한 \(h\)의 진짜 오차(true risk)
\(m_\mathcal{H}(\varepsilon,\delta)\): 표본 복잡도 — \(\varepsilon, \delta\) 달성에 필요한 최소 샘플 수
\(A(S)\): 훈련 샘플 \(S\)를 입력받은 학습 알고리즘 \(A\)의 출력 가설

PAC 프레임워크 주요 개념

개념기호의미
가설 클래스 \(\mathcal{H}\) 가능한 모든 학습 모델의 집합
진짜 오차 \(L_D(h)\) 분포 \(D\)에서 \(h\)의 오류율 (일반화 오차)
경험적 오차 \(L_S(h)\) 훈련 샘플 \(S\)에서 \(h\)의 오류율

↓경험적 위험 최소화 (ERM)

가장 직관적인 학습 알고리즘은 훈련 데이터에서 오차(경험적 위험)를 최소화하는 것입니다. 이를 ERM(Empirical Risk Minimization)이라 하며, PAC 학습의 표준 알고리즘입니다.

EMPIRICAL RISK MINIMIZATION
\[ h_S = \mathrm{ERM}_{\mathcal{H}}(S) = \underset{h\in\mathcal{H}}{\arg\min}\; L_S(h) \quad\text{where}\quad L_S(h) = \frac{1}{n}\sum_{i=1}^n \mathbf{1}[h(x_i)\neq y_i] \]
\(h_S\): 훈련 데이터 \(S\)에서 ERM으로 선택된 가설
\(L_S(h)\): 경험적 위험(empirical risk) = 훈련 오류율
\(\mathbf{1}[\cdot]\): 지시함수 — 괄호 안 조건이 참이면 1, 거짓이면 0
ERM이 PAC 학습 가능하려면 \(\mathcal{H}\)가 유한하거나 VC 차원이 유한해야 함
무한 \(\mathcal{H}\)에서 ERM의 일반화를 다루는 것이 VC 이론의 핵심

|ℋ|유한 가설 클래스의 일반화 경계

유한 \(\mathcal{H}\)에서 ERM의 일반화 경계를 Hoeffding 부등식과 Union bound로 유도합니다. 이 결과가 "표본 복잡도는 log|ℋ|에 비례한다"는 핵심 결론의 근거입니다.

GENERALIZATION BOUND FOR FINITE ℋ
\[ P\!\left[\exists\, h\in\mathcal{H}: L_D(h)-L_S(h)>\varepsilon\right] \leq |\mathcal{H}|\cdot e^{-2n\varepsilon^2} \]
\(|\mathcal{H}|\): 가설 클래스 크기
유도 방법: 각 가설 \(h\)에 Hoeffding 부등식 적용 후 Union bound로 합산
\(\delta = |\mathcal{H}|e^{-2n\varepsilon^2}\)로 놓으면 표본 복잡도 \(n \geq \frac{\log(|\mathcal{H}|/\delta)}{2\varepsilon^2}\)
결론: 더 복잡한 모델 클래스(\(|\mathcal{H}|\) 큼)는 더 많은 데이터가 필요
SAMPLE COMPLEXITY FOR FINITE ℋ
\[ m_\mathcal{H}(\varepsilon,\delta) \leq \frac{\log|\mathcal{H}|+\log(1/\delta)}{2\varepsilon^2} \]
\(\log|\mathcal{H}|\): 가설 클래스의 "비트 수" — 모델 복잡도를 측정하는 척도
\(\log(1/\delta)\): 신뢰도 요구에 따른 추가 샘플 — \(\delta\)를 줄이면 로그 비율로 증가
\(2\varepsilon^2\): 정확도 요구 — \(\varepsilon\)을 절반으로 줄이면 필요 샘플이 4배 증가
이 공식은 "log|ℋ| bits의 설명 길이를 가진 모델 클래스는 O(log|ℋ|/ε²) 샘플로 PAC 학습 가능"을 의미

⚖Bias-Variance에서 PAC으로

PAC 이론은 Bias-Variance tradeoff를 더 엄밀하게 설명합니다. 진짜 오차는 근사 오차(approximation error)와 추정 오차(estimation error) 두 부분으로 분해됩니다.

ESTIMATION ERROR VS APPROXIMATION ERROR
\[ L_D(h_S) \leq \underbrace{L_D(h^*)}_{\text{approximation error}} + \underbrace{L_D(h_S)-L_D(h^*)}_{\text{estimation error}} \]
\(h^*\): \(\mathcal{H}\) 내 진짜 최적 가설 — \(\arg\min_{h\in\mathcal{H}} L_D(h)\)
Approximation error: \(\mathcal{H}\)가 얼마나 좋은 모델을 포함하는가 → bias에 해당
Estimation error: 유한한 데이터로 \(h^*\)에 얼마나 가까이 갈 수 있는가 → variance에 해당
모델 복잡도↑ → approximation error↓, estimation error↑ (tradeoff 관계)
모델 복잡도 ↑ (큰 ℋ)
  • Approximation error ↓ — 좋은 가설 포함
  • Estimation error ↑ — 더 많은 샘플 필요
  • 과적합(overfitting) 위험 증가
모델 복잡도 ↓ (작은 ℋ)
  • Approximation error ↑ — 최적 가설 부재
  • Estimation error ↓ — 적은 샘플로 학습
  • 과소적합(underfitting) 위험 증가
📄

PAC 이론은 "완벽하게 분리 가능한 데이터"(Realizable) 설정에서 시작합니다. 즉 \(\exists\, h^* \in \mathcal{H}: L_D(h^*) = 0\)을 가정합니다. 실제 노이즈 있는 데이터에 대한 이론을 Agnostic PAC이라 하며, 이 경우 \(\min_{h\in\mathcal{H}} L_D(h)\)가 0보다 클 수 있습니다.

(a)   더 많은 모델을 보려면 각 모델에 대한 오차를 계산해야 하기 때문
