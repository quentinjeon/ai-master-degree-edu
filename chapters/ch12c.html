Ch.12C · LLM 정렬 (Unit 3 고도화)

RLHF·선호도 학습·DPO

"도움이 되고, 해롭지 않고, 정직한" AI를 만드는 수학 — 인간 피드백으로 LLM을 정렬하는 RLHF와 더 단순한 DPO.

🧒 초등학생 한 줄 설명

"개를 훈련시킬 때 '앉아!'라고 할 때마다 잘 앉으면 간식을 주고, 못 앉으면 안 준다. RLHF는 LLM에게 '좋은 대답을 하면 점수를 올리고, 나쁜 대답을 하면 내린다'를 수백만 번 반복하는 것이다."

  • 나쁜 답변: "폭발물 만드는 법을 가르쳐 드릴게요" → 인간 평가자가 0점
  • 좋은 답변: "그런 정보는 제공할 수 없습니다, 대신 안전한 방법으로..." → 인간 평가자가 10점
  • 보상 모델이 이 패턴을 학습 → PPO로 LLM이 높은 점수를 받도록 최적화
  • AI에서: ChatGPT, Claude, Gemini가 모두 RLHF 또는 DPO 방식으로 "정렬"됨

⚖정렬 문제 — SFT만으로는 부족하다

SFT는 "이렇게 대화해"를 가르치지만, 인간이 더 선호하는 방식의 미묘한 뉘앙스를 전달하기 어렵습니다. 정렬(Alignment)은 LLM이 인간의 의도·가치관·안전 기준에 맞게 행동하도록 하는 과정입니다.

SFT 단독으로는 다음과 같은 문제가 발생합니다: 모델이 지시는 따르지만 해로운 내용을 생성하거나, 모르는 것을 아는 척하거나, 사용자가 원하는 방식이 아닌 형식으로 대답하는 경우가 잦습니다. 이를 해결하기 위해 인간 피드백 기반 강화학습(RLHF)이 등장했습니다.

🔑 HHH 정렬 목표 (Anthropic)

Anthropic이 제시한 세 가지 정렬 원칙 — 좋은 AI 어시스턴트의 핵심 기준입니다.

  • Helpful (도움이 되는): 사용자 요청을 잘 이행하고, 실질적으로 유용한 정보를 제공
  • Harmless (해롭지 않은): 위험하거나 비윤리적인 내용을 출력하지 않음 — 물리적·심리적 피해 방지
  • Honest (정직한): 모르는 것을 안다고 하지 않음 — 불확실성을 솔직하게 표현

🏋RLHF 3단계 파이프라인

RLHF(Reinforcement Learning from Human Feedback)는 세 단계로 이루어집니다. Step 1: SFT 모델 준비 → Step 2: 보상 모델(RM) 학습 (인간 평가자가 두 응답 중 선호 선택 → Bradley-Terry 모델) → Step 3: PPO로 정책 최적화 (KL 발산으로 제한)

보상 모델 학습 — Bradley-Terry 모델 (Ziegler et al., 2019)
\[ \mathcal{L}_{\text{RM}}(\phi) = -\mathbb{E}_{(x,y_w,y_l)\sim\mathcal{D}}\!\left[\log\sigma\!\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right)\right] \]
\(r_\phi(x,y)\): 보상 모델이 (프롬프트 \(x\), 응답 \(y\)) 쌍에 부여하는 점수
\(y_w\): 인간이 선호하는 응답 (winner) — 두 응답 중 더 좋다고 선택된 응답
\(y_l\): 인간이 비선호하는 응답 (loser) — 두 응답 중 더 나쁘다고 평가된 응답
\(\sigma\): 시그모이드 함수. \(r_\phi(y_w) > r_\phi(y_l)\)이 될 때 손실이 작아짐
Bradley-Terry 모델: 두 항목 간 선호도를 점수 차이의 시그모이드로 모델링하는 고전 통계 모델
PPO 최적화 목적함수
\[ \mathcal{J}_{\text{RLHF}}(\theta) = \mathbb{E}_{x\sim\mathcal{D},\, y\sim\pi_\theta(\cdot|x)}\!\left[r_\phi(x,y) - \beta\, D_{\text{KL}}\!\left(\pi_\theta(\cdot|x) \,\|\, \pi_{\text{ref}}(\cdot|x)\right)\right] \]
\(\pi_\theta\): 현재 LLM 정책 (학습 중인 모델) — 파라미터 \(\theta\)로 정의
\(\pi_{\text{ref}}\): 기준 정책 (SFT 모델, 고정) — 정렬 전의 기초 모델
\(\beta\, D_{\text{KL}}\): KL 발산 페널티 — 보상 해킹(reward hacking) 발생 방지
\(\beta\): 정규화 강도. 높을수록 기준 모델에 가깝게 유지, 낮을수록 보상 극대화에 집중
보상 해킹: 모델이 보상 모델의 빈틈을 이용해 비정상적 텍스트로 높은 점수를 받는 현상

RLHF 단계별 요약

단계 학습 대상 데이터 목적
1. SFT LLM 전체 또는 LoRA (질문, 답변) 쌍 지시 따르기
2. RM 학습 보상 모델 (별도 LLM) (질문, 좋은응답, 나쁜응답) 선호도 점수 예측
3. PPO LLM (LoRA 권장) 프롬프트만 보상 최대화

⚡DPO — PPO의 복잡함을 수식 하나로

DPO(Direct Preference Optimization)는 RLHF의 수학을 분석하여 보상 모델 없이도 동일한 해로 수렴할 수 있음을 보인 획기적인 방법입니다. PPO의 복잡한 RL 루프를 제거하고, 선호도 데이터에서 직접 LLM을 최적화합니다.

DPO 손실함수 (Rafailov et al., 2023)
\[ \mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E}_{(x,y_w,y_l)}\!\left[\log\sigma\!\left(\beta\log\frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta\log\frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right] \]
DPO는 보상 모델 \(r_\phi\)를 로그 확률 비(log likelihood ratio)로 암묵적으로 표현
\(\log \pi_\theta(y|x) - \log \pi_{\text{ref}}(y|x)\): 암묵적 보상의 증가분
좋은 응답의 암묵적 보상 증가 > 나쁜 응답의 증가 → 손실 최소화
PPO 불필요: 보상 모델 학습, RL 루프 모두 제거 → 2023년 이후 표준 정렬 방법
\(\beta\): RLHF의 KL 페널티 강도와 동일한 역할 — 기준 모델 이탈 정도 조절
💡

DPO의 수학적 통찰: RLHF의 KL-제약 RL 문제는 closed-form 해가 존재합니다. \(\pi^*(y|x) \propto \pi_{\text{ref}}(y|x) \cdot \exp(r(x,y)/\beta)\). DPO는 이 관계를 역으로 이용해서 \(r\)을 \(\pi_\theta\)로 표현, 보상 모델(RM) 학습 없이 직접 최적화합니다.

⚖현장에서 RLHF vs DPO 선택 기준

RLHF와 DPO는 같은 목표를 다른 경로로 달성합니다. 실제 프로젝트에서는 목적과 자원에 따라 선택이 달라집니다.

항목 RLHF + PPO DPO
보상 모델 ✅ 별도 학습 필요 ❌ 불필요
학습 단계 3단계 1단계
GPU 메모리 높음 (4개 모델 동시) 낮음 (2개 모델)
안정성 낮음 (RL 불안정) 높음
표현력 높음 (온라인 최적화) 상대적으로 낮음
사용 사례 고성능 상용 LLM 연구, 도메인 특화
🏋 RLHF + PPO 적합한 경우
  • GPT-4, Claude 수준의 상용 정렬
  • 온라인 피드백으로 지속 개선 필요
  • 보상 모델 품질이 핵심인 경우
  • 대규모 인프라 보유 팀
⚡ DPO 적합한 경우
  • 연구 프로토타입, 빠른 실험
  • 도메인 특화 선호도 학습
  • GPU 자원 제한 환경
  • Llama, Mistral 등 오픈소스 파인튜닝

⚡ DPO β — 선호도 반영 강도 시각화

β와 로그 확률 비를 조절하여 DPO 손실이 어떻게 변하는지 확인하세요.

β (KL 정규화 강도)
log π(y_w)/π_ref(y_w)
log π(y_l)/π_ref(y_l)
결과가 여기에 표시됩니다.
