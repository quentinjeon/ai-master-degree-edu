Ch.12 · 딥러닝 & LLM (14주차)

딥러닝과 언어 모델 (LLM)

언어모델이 다음 단어를 확률로 고른다는 것. \(P(\text{다음 단어} \mid \text{앞 문장들})\)이 LLM의 핵심입니다.

🧒 초등학생 한 줄 설명

"문장을 읽고 '다음에 무슨 단어가 나올까?' 맞히는 게임을 엄청 많이 해서 똑똑해진 것이 LLM이다."

  • "나는 학교에 ___" → "가다" 확률 70%, "다니다" 확률 20%, "오다" 10% — 다음 단어를 확률로 선택
  • LLM은 이 예측을 조 단위 문장 데이터로 학습해 엄청나게 정교한 확률 추정기가 된다
  • AI에서: ChatGPT, Claude, LLaMA 등 모두 \(P(\text{다음 토큰} \mid \text{이전 토큰들})\)을 학습한 모델

📝언어 모델 = 조건부 확률의 연쇄

문장의 확률은 각 단어의 조건부 확률의 곱으로 표현됩니다. 이것은 확률의 연쇄 법칙(Chain Rule)을 그대로 적용한 것입니다. LLM은 각 위치에서 이전 토큰들을 조건으로 다음 토큰의 확률 분포를 추정하는 함수입니다.

문장 확률 — Autoregressive Factorization
\[ P(w_1, w_2, \ldots, w_T) = \prod_{t=1}^{T} P(w_t \mid w_1, \ldots, w_{t-1}) = \prod_{t=1}^{T} P(w_t \mid w_{ \(w_t\): t번째 토큰 — 단어, 서브워드, 문자 등 LLM의 기본 처리 단위
\(w_{<t}\): t 이전의 모든 토큰(context) — 현재까지 생성된 전체 문맥
연쇄 법칙(Chain Rule)의 직접적인 응용 — 결합 확률을 조건부 확률의 곱으로 분해
LLM 학습 = 이 조건부 확률들을 가장 잘 추정하는 파라미터 \(\theta\) 찾기
💡

Autoregressive(자기회귀) 모델: 이전 출력을 다음 입력으로 사용하며 순차적으로 토큰을 생성. GPT 계열이 이 방식을 따릅니다. 문장 전체의 확률은 각 단계 조건부 확률의 곱으로 계산됩니다.

🎯LLM 학습 목표 — 음의 로그우도 최소화

LLM은 주어진 텍스트 데이터에서 각 위치의 다음 토큰을 정확히 예측하도록 학습됩니다. 이 학습 목표는 음의 로그우도(Negative Log-Likelihood, NLL) 최소화로 표현되며, 이는 MLE(최대우도추정)와 동일합니다.

언어 모델 손실함수 (NLL)
\[ \mathcal{L} = -\frac{1}{T}\sum_{t=1}^{T} \log P_\theta(w_t \mid w_{ 음의 로그우도(NLL) = 언어모델의 표준 손실함수 — 이것을 최소화하는 방향으로 학습
이것을 최소화 = 각 위치에서 다음 토큰을 잘 맞추는 것 = MLE(최대우도추정)
\(\theta\): LLM의 파라미터(가중치) — 수십억~수조 개의 실수값
Perplexity: \(\exp(\mathcal{L})\) — 낮을수록 좋은 언어모델, 언어모델 성능 평가의 표준 지표
🔑 NLL ↔ Cross-Entropy ↔ MLE

언어모델 NLL = Cross-Entropy Loss = 음의 로그우도 최소화 = MLE.

딥러닝 분류기의 CE Loss와 본질적으로 같은 목적함수. LLM 학습의 수학적 본질은 거대한 MLE다.

🎰토큰 샘플링 전략 — Temperature, Top-k, Top-p

LLM이 다음 토큰을 선택하는 방법을 제어하는 파라미터들입니다. 모델은 각 토큰에 대한 로짓(logit) 점수를 계산하고, 이를 다양한 전략으로 확률 분포로 변환한 뒤 샘플링합니다.

Temperature Scaling
\[ P(w_t = k \mid w_{ \(\tau\)(temperature) = 출력 분포의 "날카로움" 조절 파라미터
\(\tau \to 0\): 항상 가장 높은 확률 토큰 선택(greedy, 결정론적) — 분포가 one-hot에 가까워짐
\(\tau = 1\): 원래 softmax 분포 그대로
\(\tau > 1\): 분포를 부드럽게(더 다양한 출력, 창의적) — 낮은 확률 토큰도 선택될 기회 증가
전략설명특징사용 시
Greedy항상 확률 최대 토큰 선택빠르지만 반복적·단조로움정확한 사실 답변
Temperature (τ)분포 날카로움 조절τ 높으면 창의적, 낮으면 보수적창의적 생성
Top-k상위 k개 토큰만 샘플링품질/다양성 균형일반적 대화
Top-p (nucleus)누적확률 p까지만 포함동적 k 결정 — 상황에 따라 후보 수 변동고품질 생성
💡

Top-p(Nucleus Sampling): 확률이 높은 토큰부터 누적합이 p(예: 0.9)에 도달할 때까지의 토큰 집합에서만 샘플링합니다. 분포가 날카로울 때는 적은 후보를, 분포가 평탄할 때는 많은 후보를 자동으로 선택합니다.
