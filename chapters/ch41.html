🔥PyTorch 핵심 패턴 — 실험 구현의 최소 단위

PyTorch는 연구 구현의 사실상 표준입니다. 전체 라이브러리를 다 알 필요는 없습니다. 논문 실험을 구현하는 데 필요한 핵심 5개 패턴만 익히면 됩니다.

PyTorch 핵심 5패턴

패턴역할핵심 코드
Dataset + DataLoader데이터 로딩 파이프라인DataLoader(dataset, batch_size=32, shuffle=True)
nn.Module모델 정의class Model(nn.Module): def forward(self, x): ...
Optimizer + Loss학습 루프loss.backward(); optimizer.step()
Checkpoint 저장모델 저장·불러오기torch.save({'state': model.state_dict()}, path)
device 관리GPU/CPU 자동화device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
PyTorch 학습 루프 — 수식과 코드의 대응
\[ \theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta \mathcal{L}(\theta_t;\, \mathcal{B}_t) \]
\(\eta\): 학습률 (learning rate) → config.yaml: lr: 0.001
\(\nabla_\theta \mathcal{L}\): 손실 그라디언트 → loss.backward()
\(\theta_{t+1} = \theta_t - \eta\nabla\mathcal{L}\): 파라미터 업데이트 → optimizer.step()
\(\mathcal{B}_t\): 미니배치 → DataLoader가 자동으로 제공
이 한 줄의 수식이 PyTorch 학습 루프 전체를 표현한다
✅

Dataset 클래스를 먼저 분리하라. 전처리 로직이 모델 코드와 섞이면 나중에 데이터를 바꿀 때 코드 전체를 수정해야 한다. class MyDataset(Dataset)으로 독립 파일을 만드는 것이 재현성의 시작이다.

⚙Config 파일 — 하드코딩을 없앤다

코드에 숫자를 직접 쓰는 것(하드코딩)은 실험 관리의 최대 실수입니다. Learning rate, batch size, epoch 수 등 모든 파라미터는 config 파일에서 관리해야 합니다.

❌ 하드코딩
  • lr = 0.001 (코드 내부)
  • for i in range(100):
  • 파라미터 파악이 불가
  • 코드 수정 → 다시 실행 필요
✅ Config 관리
  • config.yaml: lr: 0.001
  • config.yaml: epochs: 100
  • config 파일 하나로 실험 복원
  • config만 바꾸고 실행
Config YAML 표준 구조
\[ \texttt{config.yaml:} \quad \begin{cases} \texttt{model: \{ hidden\_dim: 256,\ num\_layers: 4 \}} \\ \texttt{train: \{ lr: 0.001,\ epochs: 100,\ batch\_size: 32 \}} \\ \texttt{seed: 42} \\ \texttt{data: \{ dataset: "squad",\ split: 0.8 \}} \end{cases} \]
Python에서 import yaml; cfg = yaml.safe_load(open('config.yaml')) 로 dict로 읽음
각 실험은 별도 config 파일로 → exp01.yaml, exp02.yaml
논문 Experiment Setting 섹션 = config 파일의 핵심 파라미터 목록
Git에 config 파일을 커밋하면 실험 이력이 자동 기록됨

🌱Seed 고정 — 재현 가능한 결과의 기초

딥러닝 실험은 가중치 초기화, 데이터 셔플, 드롭아웃 등에서 난수를 사용합니다. Seed를 고정하지 않으면 같은 코드로도 매번 다른 결과가 나옵니다. SCI 논문은 "동일 seed에서 재현 가능"해야 합니다.

Seed 고정 표준 코드 패턴
\[ \texttt{seed\_everything}(42): \quad \begin{aligned} &\texttt{random.seed}(42) \\ &\texttt{np.random.seed}(42) \\ &\texttt{torch.manual\_seed}(42) \\ &\texttt{torch.cuda.manual\_seed\_all}(42) \\ &\texttt{torch.backends.cudnn.deterministic = True} \end{aligned} \]
Seed 42: 관습적으로 가장 많이 쓰이는 값 (통계적 의미는 없음)
논문에서 "We use seed 42 for reproducibility"처럼 반드시 명시
단일 seed보다 여러 seed 평균이 더 신뢰도 높음: seed [42, 123, 456] 3회 실험
논문 보고 형식: "85.3 ± 0.4%" (평균 ± 표준편차)
⚠️

Seed 고정만으로 완전한 재현성이 보장되지 않는다. CUDA 비결정론적 연산, 병렬 처리 순서, 라이브러리 버전 차이 등이 결과에 영향을 줄 수 있다. 논문에서는 seed + 환경(Python/PyTorch/CUDA 버전)을 함께 공개하는 것이 표준이다.
