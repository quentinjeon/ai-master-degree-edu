확률 (Probability)

어떤 일이 일어날 가능성을 숫자로 말하는 것

▼
🧒 초등학생 버전

확률은 "어떤 일이 일어날 가능성이 얼마나 큰지 숫자로 말하는 것"이다.

  • 동전을 던졌을 때 앞면이 나올 가능성 → 50%
  • 내일 비가 올 가능성 → 30%
  • 시험에서 100점 맞을 가능성 → 모름, 하지만 숫자로 표현 가능
🤖 AI에서 왜 중요해?

AI는 "정답 100%"를 항상 말하지 못한다. 대신 이렇게 생각한다:

  • 고양이일 가능성 90%
  • 강아지일 가능성 10%

즉, AI는 확률로 생각한다.

📐 수식으로는

\(0 \le P(A) \le 1\)

P(앞면) = 0.5

P(비가 옴) = 0.3

확실한 일: P = 1

불가능한 일: P = 0

🔍

조건부확률 (Conditional Probability)

어떤 사실을 이미 알고 있을 때의 확률

▼
🧒 초등학생 버전

조건부확률은 "어떤 사실을 이미 알고 있을 때, 다른 일이 일어날 가능성"이다.

  • 그냥 비 올 확률: 30%
  • 하늘이 엄청 검다는 걸 알게 됨 → 비 올 확률이 올라감: 80%
  • 새 정보가 생기면 생각이 바뀐다!
🤖 AI에서 왜 중요해?
  • 그냥 문장만 보면 답변 A일 것 같음
  • 이전 대화 내용을 알게 됨
  • → 답변 확률이 달라짐

LLM은 P(다음 단어 | 앞 문장들)을 계산한다.

📐 수식으로는

\( P(A \mid B) = \dfrac{P(A \cap B)}{P(B)} \)

A|B = "B가 일어났을 때 A가 일어날 확률"

표본공간이 B로 줄어들었다고 생각하면 됨

베이즈 정리 (Bayes' Theorem)

새 정보를 얻고 나서 내 생각을 업데이트하는 공식

▼
🧒 초등학생 버전

베이즈 정리는 "새로운 정보를 얻고 나서 내 생각을 업데이트하는 공식"이다.

  • 어떤 병은 매우 드묾 (처음 생각: prior)
  • 검사 결과가 양성 나옴 (새 정보: likelihood)
  • 하지만 검사기도 틀릴 수 있음!
  • → 진짜 병일 가능성을 다시 계산 (업데이트된 생각: posterior)

"처음 생각" + "새 정보" = "업데이트된 생각"

🤖 AI에서 왜 중요해?
  • prior: 보기 전 믿음
  • likelihood: 데이터가 나올 확률
  • posterior: 데이터 본 후 믿음

베이지안 딥러닝, 스팸 필터, 의료 AI에서 핵심으로 사용

📐 수식으로는

\( P(\theta \mid D) = \dfrac{P(D \mid \theta) \cdot P(\theta)}{P(D)} \)

posterior ∝ likelihood × prior

확률변수 (Random Variable)

랜덤하게 일어나는 일을 숫자로 바꾼 것

▼
🧒 초등학생 버전

확률변수는 랜덤하게 일어나는 일을 숫자로 바꾼 것이다.

  • 동전 앞면 = 1, 뒷면 = 0 → 계산 가능!
  • 주사위 결과 = 1, 2, 3, 4, 5, 6 → 평균, 분산 계산 가능!
  • 원래는 "사건"인데, 계산하려고 숫자로 바꾸는 것
🤖 AI에서 왜 중요해?

AI는 글이나 이미지도 결국 숫자로 바꿔서 계산한다.

단어 → 숫자 (token ID)

이미지 → 숫자 배열 (픽셀값)

→ 모든 것이 확률변수다

📐 이산형 vs 연속형

이산형: 셀 수 있는 값

예: 주사위, 클래스 레이블

연속형: 연속적인 값

예: 온도, 가격, 픽셀 강도

📊

확률분포 (Probability Distribution)

어떤 값이 얼마나 자주 나오는지 보여주는 지도

▼
🧒 초등학생 버전

분포는 "어떤 값이 얼마나 자주 나오는지 보여주는 지도" 같은 것이다.

  • 주사위: 1~6이 비슷하게 나옴 (균등분포)
  • 시험 점수: 중간 점수가 많고, 너무 높거나 낮은 점수는 적음 (정규분포)
  • 데이터에는 보통 어떤 "모양"이 있다
🤖 AI에서 왜 중요해?
  • Bernoulli: 이진 분류 출력
  • Categorical: 다중 분류 (softmax)
  • Gaussian: 회귀 출력, 노이즈 모델링

그 모양을 이해해야 AI를 잘 만들 수 있다

📐 주요 분포

\(\mathcal{N}(\mu, \sigma^2)\): 정규분포

\(\text{Bern}(p)\): 베르누이

\(\text{Bin}(n,p)\): 이항분포

\(\text{Cat}(\mathbf{p})\): 카테고리컬

기대값 (Expected Value)

많이 반복했을 때 평균적으로 어느 정도가 나오는지

▼
🧒 초등학생 버전

기대값은 많이 많이 반복했을 때 평균적으로 어느 정도가 나오는지다.

  • 동전을 1000번 던지면 앞면은 약 500번 → 기대값 0.5
  • 주사위를 1000번 던지면 평균 눈은 약 3.5 → 기대값 3.5
🤖 AI에서 왜 중요해?

AI 학습도 결국 "평균적으로 얼마나 잘 맞추는지"를 본다.

Loss = \(\mathbb{E}[\mathcal{L}]\) — 기대 손실을 최소화

Gradient도 기대값으로 표현됨

📐 수식으로는

\(\mathbb{E}[X] = \sum_x x \cdot P(X=x)\)

(이산형)

\(\mathbb{E}[X] = \int x \, f(x) \, dx\)

(연속형)

📏

분산 (Variance)

값들이 얼마나 흩어져 있는지

▼
🧒 초등학생 버전

분산은 값들이 얼마나 흩어져 있는지를 말한다.

  • 학생 5명 키가 거의 비슷 → 분산 작음
  • 어떤 학생은 아주 작고 어떤 학생은 아주 큼 → 분산 큼
🤖 AI에서 왜 중요해?

AI가 어떤 문제에서는 자신 있게, 어떤 문제에서는 흔들릴 수 있다.

그 흔들림 = 분산 (불확실성)

Bias-Variance Tradeoff의 핵심 개념

📐 수식으로는

\(\text{Var}(X) = \mathbb{E}[(X - \mu)^2]\)

= \(\mathbb{E}[X^2] - (\mathbb{E}[X])^2\)

표준편차 \(\sigma = \sqrt{\text{Var}(X)}\)

우도 (Likelihood)

이 데이터가 나왔다면, 어떤 설명이 가장 그럴듯한가?

▼
🧒 초등학생 버전

likelihood는 "이 데이터가 나왔다면, 어떤 설명이 가장 그럴듯한가?"를 보는 것이다.

  • 동전을 10번 던졌는데 9번 앞면
  • "이 동전은 공평하다" → 별로 그럴듯하지 않음
  • "이 동전은 앞면이 잘 나온다" → 더 그럴듯함!
🤖 Probability vs Likelihood

확률: 파라미터 고정, 데이터 변함
P(데이터 | θ 고정)

우도: 데이터 고정, 파라미터 변함
L(θ | 데이터 고정)

📐 수식으로는

\( \mathcal{L}(\theta \mid \mathcal{D}) = P(\mathcal{D} \mid \theta) \)

데이터가 주어졌을 때 θ의 함수

🎯

최대우도추정 (MLE)

데이터를 가장 잘 설명하는 답을 고르는 방법

▼
🧒 초등학생 버전

MLE는 "데이터를 가장 잘 설명하는 답을 고르는 방법"이다.

  • 발자국을 보고 "누가 지나갔을까?" 추리하는 것
  • 시험 결과를 보고 "이 학생 평균 실력은?" 찾는 것
  • 가장 그럴듯한 값을 찾는 것
🤖 AI에서 왜 중요해?

딥러닝 훈련의 핵심!

Cross-Entropy 손실을 최소화 = 음의 로그우도 최소화 = MLE

즉, 딥러닝 학습은 MLE다.

📐 수식으로는

\( \hat{\theta}_{MLE} = \argmax_\theta \sum_{i=1}^n \log p(x_i \mid \theta) \)

곱 → 합 변환(로그)으로 계산 편리

🌀

엔트로피 (Entropy)

얼마나 헷갈리는지, 얼마나 불확실한지를 숫자로

▼
🧒 초등학생 버전

entropy는 얼마나 헷갈리는지, 얼마나 불확실한지를 숫자로 나타낸 것이다.

  • 동전 앞/뒤가 딱 반반 → 결과가 헷갈림 → entropy 큼
  • 항상 앞면만 나오는 동전 → 전혀 안 헷갈림 → entropy = 0
🤖 AI에서 왜 중요해?

AI가 여러 답 중 무엇이 맞는지 헷갈릴 때 entropy가 커진다.

불확실성 측정, 정보량 계산의 기초

결정 트리 분기 기준으로도 사용

📐 수식으로는

\( H(P) = -\sum_x P(x) \log P(x) \)

균등분포일 때 최대

확실할수록 (P=1인 값) 최소 = 0

🏹

크로스 엔트로피 (Cross-Entropy)

정답과 AI의 예상이 얼마나 다른지 재는 벌점표

▼
🧒 초등학생 버전

cross entropy는 정답과 AI의 예상이 얼마나 다른지 재는 벌점표다.

  • 정답은 고양이
  • AI가 고양이 99% → 벌점 아주 작음
  • AI가 고양이 5% → 벌점 아주 큼
🤖 AI에서 왜 중요해?

딥러닝 분류 문제에서 가장 많이 쓰는 loss다.

CE를 줄이는 방향으로 학습하면 AI가 점점 정답에 가까워진다.

CE 최소화 = MLE와 동치!

📐 수식으로는

\( H(P, Q) = -\sum_x P(x) \log Q(x) \)

P: 정답분포, Q: 모델예측분포

H(P,Q) = H(P) + D_{KL}(P||Q)

↔️

KL 발산 (KL Divergence)

두 생각이 얼마나 다른지 재는 도구

▼
🧒 초등학생 버전

KL divergence는 "두 생각이 얼마나 다른지" 재는 도구다.

  • 진짜 정답 분포와 모델이 믿는 분포가 얼마나 다른지 본다
  • 같으면 0, 다를수록 커짐
🤖 AI에서 왜 중요해?

VAE, GAN, RLHF 등 생성 모델 논문에서 핵심으로 등장

두 분포의 차이를 줄이는 방향으로 학습

📐 수식으로는

\( D_{KL}(P \| Q) = \sum_x P(x) \log \dfrac{P(x)}{Q(x)} \)

항상 ≥ 0, P=Q일 때만 0

비대칭: D(P‖Q) ≠ D(Q‖P)

🌡️

캘리브레이션 (Calibration)

AI가 말하는 자신감이 진짜 맞는지 확인하는 것

▼
🧒 초등학생 버전

calibration은 AI가 말하는 자신감이 진짜 맞는지 확인하는 것이다.

  • AI가 "나 90% 확신해"라고 100번 말했는데
  • 진짜 90번 정도 맞으면 → calibration 좋음 ✅
  • 60번만 맞으면 → calibration 나쁨 ❌
  • 자신만만하게 틀리는 AI가 제일 위험하다!
🤖 AI에서 왜 중요해?

의료, 금융, 자율주행 등 실서비스에서 매우 중요

"90% 확신"인데 실제로 60%만 맞으면 큰 문제

논문에서 ECE(Expected Calibration Error)로 측정

📐 수식으로는

잘 calibrated된 모델:

\( P(\hat{y} = y \mid \hat{p} = p) = p \)

신뢰도 p라고 할 때, 실제로 p의 비율로 맞아야 함

집합과 함수

수학적 글쓰기의 기본 언어를 배웁니다. 논문에서 등장하는 수학 기호를 읽을 수 있게 됩니다.

🧒 초등학생 한 줄 설명

집합은 "같은 특징을 가진 것들의 묶음"이고, 함수는 "뭔가를 넣으면 뭔가 나오는 기계"다. 수학 기호는 어려운 마법이 아니라 짧게 쓰는 약속이다.

∈집합 (Set)

집합은 뚜렷이 구별되는 원소들의 모임입니다. AI 논문에서 데이터셋, 파라미터 공간, 클래스 레이블 등을 집합으로 표현합니다.

AI 논문에서의 집합 표기
\[ \mathcal{D} = \{(\mathbf{x}_1, y_1), (\mathbf{x}_2, y_2), \ldots, (\mathbf{x}_n, y_n)\} \]
훈련 데이터셋 \(\mathcal{D}\)는 입력-출력 쌍의 집합. \(\mathcal{X} = \mathbb{R}^d\): d차원 입력 공간

핵심 기호 표

기호읽는 법의미예시
\(x \in A\)x는 A에 속한다x는 집합 A의 원소\(3 \in \{1,2,3\}\)
\(A \subseteq B\)A는 B에 포함된다A는 B의 부분집합\(\{1,2\} \subseteq \{1,2,3\}\)
\(A \cup B\)A 합집합 BA 또는 B에 속하는 원소\(\{1,2\} \cup \{2,3\} = \{1,2,3\}\)
\(A \cap B\)A 교집합 BA이면서 B인 원소\(\{1,2\} \cap \{2,3\} = \{2\}\)
\(\forall x\)모든 x에 대해for all x\(\forall x \in \mathbb{R},\ x^2 \ge 0\)
\(\exists x\)어떤 x가 있어서there exists x\(\exists x : f(x) = 0\)
\(\mathbb{R}^d\)d차원 실수 공간d개 실수의 벡터 공간\(\mathbf{x} \in \mathbb{R}^{768}\) (BERT)

f함수 (Function)

함수 \(f: X \to Y\)는 정의역 \(X\)의 각 원소를 공역 \(Y\)의 정확히 하나의 원소에 대응시키는 규칙입니다.

🔑 핵심 개념 — 머신러닝 모델은 함수다

모든 AI 모델은 수학적으로 함수다:

\[ f_\theta : \mathcal{X} \to \mathcal{Y} \]

파라미터 \(\theta\)를 가진 모델이 입력 \(\mathbf{x} \in \mathcal{X}\)에서 출력 \(\hat{y} \in \mathcal{Y}\)으로 매핑한다.

딥러닝의 함수 합성
\[ f(\mathbf{x}) = f_L \circ f_{L-1} \circ \cdots \circ f_1(\mathbf{x}) \]
L개 레이어를 가진 신경망은 L개 함수의 합성이다. \(\circ\)는 합성 연산자.
✅

논문에서 자주 보이는 표현: \(\argmin_\theta \mathcal{L}(\theta)\) — "손실함수 \(\mathcal{L}\)을 최소화하는 파라미터 \(\theta\)를 찾아라"

미분과 최적화

AI 학습의 엔진인 경사하강법(Gradient Descent)을 이해하기 위한 핵심 개념입니다.

🧒 초등학생 한 줄 설명

미분은 "조금 변할 때 얼마나 달라지는지"다. 경사하강법은 "산에서 제일 빠른 길로 내려오는 것"처럼 AI가 실수를 줄여나가는 방법이다.

∂미분 (Derivative)

미분은 함수의 순간 변화율입니다. "입력이 조금 변할 때 출력이 얼마나 변하는가?"

미분의 정의
\[ f'(x) = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h} \]

기본 미분 공식

함수 \(f(x)\)미분 \(f'(x)\)AI 활용
\(x^n\)\(nx^{n-1}\)다항식 층
\(e^x\)\(e^x\)Softmax 분모
\(\ln x\)\(\dfrac{1}{x}\)Log-likelihood, CE loss
\(\sigma(x) = \dfrac{1}{1+e^{-x}}\)\(\sigma(x)(1-\sigma(x))\)시그모이드 역전파
\(\text{ReLU}(x) = \max(0,x)\)\(\mathbf{1}[x>0]\)딥러닝 활성화 함수

∇그레디언트 (Gradient)

여러 파라미터가 있을 때 각각에 대한 편미분을 벡터로 모은 것이 그레디언트입니다.

그레디언트 벡터
\[ \nabla_\theta \mathcal{L} = \left( \frac{\partial \mathcal{L}}{\partial \theta_1}, \frac{\partial \mathcal{L}}{\partial \theta_2}, \ldots, \frac{\partial \mathcal{L}}{\partial \theta_n} \right)^\top \]
\(\nabla_\theta \mathcal{L}\)은 손실함수가 가장 빠르게 증가하는 방향. 반대 방향이 하강 방향.

⬇경사하강법 (Gradient Descent)

파라미터 업데이트 규칙
\[ \theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta \mathcal{L}(\theta_t) \]
\(\eta\) (학습률): 너무 크면 발산, 너무 작으면 느리게 수렴. 보통 0.001~0.1 사용.

🔢 경사하강법 시뮬레이터 — \(f(x)=x^2\)

간단한 함수 \(f(x) = x^2\)에서 경사하강법을 체험. \(f'(x) = 2x\)이므로 \(x_{t+1} = x_t - \eta \cdot 2x_t\)

시작점 \(x_0\)
학습률 \(\eta\)
반복 횟수
결과가 여기에 표시됩니다.
⚠️

연쇄 법칙 (Chain Rule): 역전파(Backpropagation)의 핵심.
\(\dfrac{\partial \mathcal{L}}{\partial x} = \dfrac{\partial \mathcal{L}}{\partial z} \cdot \dfrac{\partial z}{\partial x}\) — 합성 함수를 거꾸로 미분하며 그레디언트 전달

선형대수 기초

딥러닝에서 데이터와 모델 파라미터는 벡터와 행렬로 표현됩니다.

🧒 초등학생 한 줄 설명

벡터는 "여러 숫자를 순서대로 묶은 것"이고, 행렬은 "숫자를 격자 모양으로 배열한 것"이다. 딥러닝의 모든 계산은 이것으로 이루어진다.

→벡터 (Vector)

열벡터 표기
\[ \mathbf{x} = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_d \end{bmatrix} \in \mathbb{R}^d \]
d개의 특성을 가진 하나의 데이터 샘플. 굵은 소문자로 표기.

내적 (Dot Product)

내적 — Attention의 핵심
\[ \mathbf{a} \cdot \mathbf{b} = \mathbf{a}^\top \mathbf{b} = \sum_{i=1}^{d} a_i b_i = \|\mathbf{a}\|\|\mathbf{b}\|\cos\theta \]
두 벡터가 비슷한 방향이면 내적이 크다 (cos θ ≈ 1). Transformer Attention에서 Q·Kᵀ 연산의 기초.

⊞행렬 (Matrix)

선형 레이어 = 행렬 곱
\[ \mathbf{h} = \sigma(\mathbf{W}\mathbf{x} + \mathbf{b}) \]
\(\mathbf{W} \in \mathbb{R}^{d_{out} \times d_{in}}\): 가중치 행렬, \(\mathbf{b}\): 편향 벡터, \(\sigma\): 활성화 함수
연산표기AI 역할
전치 행렬\(\mathbf{A}^\top\)차원 변환, Attention QKᵀ
행렬 곱\(\mathbf{A}\mathbf{B}\)레이어 순전파 (forward pass)
역행렬\(\mathbf{A}^{-1}\)최소제곱법 해석적 풀이
노름 (L2)\(\|\mathbf{x}\|_2\)정규화, 거리 계산, L2 정규화
고유값 분해\(\mathbf{A}=\mathbf{Q}\mathbf{\Lambda}\mathbf{Q}^\top\)PCA, 주성분 분석
SVD\(\mathbf{A}=\mathbf{U}\mathbf{\Sigma}\mathbf{V}^\top\)LoRA, 차원 축소
🔑 논문에서 자주 보이는 표현

\(\mathbf{X} \in \mathbb{R}^{n \times d}\) — n개 샘플, d개 특성의 데이터 행렬

\(\mathbf{W} \in \mathbb{R}^{d_{out} \times d_{in}}\) — 선형 레이어의 가중치 행렬

확률론 기초

불확실성을 수학적으로 표현하는 언어. AI 모델의 예측은 본질적으로 확률입니다.

🧒 초등학생 한 줄 설명

확률은 "어떤 일이 일어날 가능성을 0부터 1 사이의 숫자로 나타낸 것"이다. 1이면 무조건 일어나고, 0이면 절대 안 일어난다.

🎲확률의 3가지 공리

Kolmogorov 공리 (모든 확률의 기초)
\[ \text{1. } P(A) \ge 0 \qquad \text{2. } P(\Omega) = 1 \qquad \text{3. } P(A \cup B) = P(A) + P(B) \text{ (A,B 배반 시)} \]

|조건부확률

🧒 초등학생 버전

비가 올 확률이 30%였는데, 갑자기 먹구름을 봤다. 그러면 비 올 확률이 80%로 올라간다. 새 정보가 생기면 확률이 바뀐다!

조건부확률 공식
\[ P(A \mid B) = \frac{P(A \cap B)}{P(B)}, \quad P(B) > 0 \]
B가 일어났음을 알게 됐을 때, A가 일어날 확률. 표본공간이 Ω에서 B로 줄어든다.

🔄베이즈 정리

🧒 초등학생 버전

"처음 생각(prior)" + "새 정보(likelihood)" = "업데이트된 생각(posterior)"

  • 어떤 병은 매우 드묾 → prior 낮음
  • 검사 결과 양성 → 새 정보
  • 하지만 검사기도 틀릴 수 있음! → posterior 계산 필요
베이즈 정리
\[ \underbrace{P(\theta \mid \mathcal{D})}_{\text{posterior}} = \frac{\overbrace{P(\mathcal{D} \mid \theta)}^{\text{likelihood}} \cdot \overbrace{P(\theta)}^{\text{prior}}}{\underbrace{P(\mathcal{D})}_{\text{evidence}}} \]
evidence P(D)는 모든 가능한 θ에 대해 적분한 값 — 정규화 상수 역할. posterior ∝ likelihood × prior
용어의미비유AI 예시
Prior \(P(\theta)\)데이터 보기 전 믿음병원 가기 전 내 생각파라미터 초기 분포
Likelihood \(P(D|\theta)\)θ일 때 D가 나올 확률이 동전으로 9번 앞면 나올 가능성데이터 생성 확률
Posterior \(P(\theta|D)\)D를 보고 업데이트된 믿음검사 결과 본 후 내 생각학습 후 파라미터 분포
Evidence \(P(D)\)정규화 상수기준점보통 계산 생략, ∝ 사용

🧒 초등학생 한 줄 설명

통계적 추론은 "일부 데이터를 보고 전체에 대해 결론을 내리는 것"이다. p-value는 "이게 그냥 우연일 가능성"이다. 낮을수록 우연이 아니다!

🎯최대우도추정 (MLE)

🧒 초등학생 버전

발자국을 보고 "누가 지나갔을까?" 추리하는 것. 가장 그럴듯한 설명(파라미터)을 찾는다.

MLE 목적함수
\[ \hat{\theta}_{MLE} = \argmax_\theta \prod_{i=1}^{n} p(x_i \mid \theta) = \argmax_\theta \sum_{i=1}^{n} \log p(x_i \mid \theta) \]
곱 → 합(로그 변환)으로 수치 안정성 확보. 딥러닝의 Cross-Entropy 최소화 = 음의 로그우도(NLL) 최소화 = MLE

🔬가설 검정

1

귀무가설 \(H_0\): "두 방법 사이에 차이가 없다" — 기존 주장

2

대립가설 \(H_1\): "새로운 방법이 더 좋다" — 내가 주장하고 싶은 것

3

검정 통계량 계산: 데이터로부터 t-통계량, z-통계량 등 계산

4

p-value 계산: \(p < \alpha = 0.05\)이면 귀무가설 기각 → "통계적으로 유의함"

🔑 p-value의 정확한 의미

p-value는 "귀무가설이 사실이라고 가정했을 때, 현재 관측된 데이터(또는 더 극단적인 결과)가 나올 확률"이다.

p = 0.03 → H₀이 사실이면 이런 데이터가 나올 확률이 3%. → 이 데이터는 H₀와 맞지 않는다 → H₀ 기각

⚠️

흔한 오해: p-value는 "가설이 옳을 확률"이 아닙니다! 또한 p < 0.05가 실용적 중요성을 보장하지 않습니다. 표본이 충분히 크면 의미 없는 차이도 유의해집니다.

📏신뢰구간 (Confidence Interval)

95% 신뢰구간
\[ \bar{x} \pm 1.96 \cdot \frac{s}{\sqrt{n}} \]
"동일한 방법으로 100번 실험하면, 95번은 이 구간이 참 모평균을 포함한다"는 의미.

🧒 초등학생 한 줄 설명

AI 학습은 "틀린 정도(loss)를 줄이는 방향으로 파라미터를 조금씩 바꾸는 것"이다. 이것이 수학적으로는 MLE + 경사하강법이다.

🧩머신러닝의 통합 공식

머신러닝의 핵심 목적함수
\[ \hat{\theta} = \argmin_\theta \underbrace{\frac{1}{n}\sum_{i=1}^{n} \mathcal{L}(f_\theta(\mathbf{x}_i), y_i)}_{\text{경험적 위험 (훈련 손실)}} + \underbrace{\lambda \Omega(\theta)}_{\text{정규화항}} \]
• \(\mathcal{L}\): 손실함수 (미분 필요)  |  • \(f_\theta\): 모델 (선형대수)  |  • \(\Omega\): 정규화 (과적합 방지)  |  • \(\lambda\): 하이퍼파라미터

📊정보이론 — Entropy, CE, KL

세 가지 관계
\[ H(P) = -\sum_x P(x)\log P(x) \qquad H(P,Q) = -\sum_x P(x)\log Q(x) \qquad D_{KL}(P\|Q) = H(P,Q) - H(P) \]
H(P): 정답의 불확실성  |  H(P,Q): 정답 분포 P를 Q로 코딩할 때 평균 길이  |  KL: 두 분포의 차이
개념초등학생 설명수식AI 역할
Entropy얼마나 헷갈리는지\(H(P)\)불확실성 측정, 결정트리
Cross-EntropyAI 예측이 정답과 얼마나 다른지 벌점\(H(P,Q)\)분류 손실함수 (CE loss)
KL Divergence두 생각이 얼마나 다른지\(D_{KL}(P\|Q)\)VAE, RLHF, 지식 증류
